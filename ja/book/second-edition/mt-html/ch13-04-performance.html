Calculations in applications like audio decoders often prioritize performance most highly.オーディオデコーダなどのアプリケーションでの計算では、パフォーマンスの優先順位が最も高くなることがよくあります。 Here, we're creating an iterator, using two adaptors, and then consuming the value.ここでは、2つのアダプタを使用し、値を消費するイテレータを作成しています。 What assembly code would this Rust code compile to?このRustコードはどのようなアセンブリコードにコンパイルされますか？ Well, as of this writing, it compiles down to the same assembly you'd write by hand.まあ、この執筆時点では、手作業で書くと同じアセンブリにコンパイルされます。 There's no loop at all corresponding to the iteration over the values in coefficients : Rust knows that there are 12 iterations, so it “unrolls” the loop. coefficientsの値に対する反復に対応するループは全くありません.Rustは反復が12回あることを知っているので、ループを "展開"します。 Unrolling is an optimization that removes the overhead of the loop controlling code and instead generates repetitive code for each iteration of the loop. アンロールとは、ループ制御コードのオーバーヘッドを取り除き、代わりにループの反復ごとに反復コードを生成する最適化です。

Summary概要


Closures and iterators are Rust features inspired by functional programming language ideas.クロージャとイテレータは、関数型プログラミング言語のアイデアに触発されたRustの機能です。 They contribute to Rust's capability to clearly express high-level ideas at low-level performance.彼らは低レベルのパフォーマンスで高レベルのアイデアを明確に表現するRustの能力に貢献します。 The implementations of closures and iterators are such that runtime performance is not affected.クロージャとイテレータの実装は、実行時のパフォーマンスに影響を与えないようなものです。 This is part of Rust's goal to strive to provide zero-cost abstractions.これはゼロコストの抽象化を提供しようとするRustの目標の一部です。
