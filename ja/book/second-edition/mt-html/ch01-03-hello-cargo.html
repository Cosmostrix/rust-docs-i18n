Hello, Cargo!こんにちは、貨物！


Cargo is Rust's build system and package manager. CargoはRustのビルドシステムとパッケージマネージャーです。 Most Rustaceans use this tool to manage their Rust projects because Cargo handles a lot of tasks for you, such as building your code, downloading the libraries your code depends on, and building those libraries.ほとんどの錆びた人はこのツールを使ってRustプロジェクトを管理しています.Cargoはコードの構築、コードが依存するライブラリのダウンロード、それらのライブラリの構築など、多くのタスクを処理するためです。 (We call libraries your code needs dependencies .) （私たちはあなたのコードに依存関係が必要なライブラリを呼び出します）。


The simplest Rust programs, like the one we've written so far, don't have any dependencies.これまでに書いたような最も単純なRustプログラムは、依存関係を持たない。 So if we had built the Hello, world!だから、こんにちは、世界を作りました！ project with Cargo, it would only use the part of Cargo that handles building your code. Cargoプロジェクトでは、コードの作成を担当するCargoの部分のみを使用します。 As you write more complex Rust programs, you'll add dependencies, and if you start a project using Cargo, adding dependencies will be much easier to do.もっと複雑なRustプログラムを書くと、依存関係が追加され、Cargoを使用してプロジェクトを開始すると、依存関係を追加する方がはるかに簡単になります。


Because the vast majority of Rust projects use Cargo, the rest of this book assumes that you're using Cargo too. Rustプロジェクトの大部分はCargoを使用しているため、この本の残りの部分では、Cargoも使用していると仮定しています。 Cargo comes installed with Rust if you used the official installers discussed in the “Installation” section. 「インストール」セクションで説明した公式のインストーラを使用した場合、CargoにはRustがインストールされます。 If you installed Rust through some other means, check whether Cargo is installed by entering the following into your terminal:他の方法でRustをインストールした場合、Cargoがインストールされているかどうかは、端末に次のように入力して確認してください。

Building for Releaseリリースのためのビルディング


When your project is finally ready for release, you can use cargo build --release to compile it with optimizations.あなたのプロジェクトが最終的にリリース準備が整ったら、 cargo build --releaseを使って最適化でコンパイルcargo build --releaseことができます。 This command will create an executable in target/release instead of target/debug .このコマンドは、 target / debugの代わりにtarget / releaseに実行可能ファイルを作成します。 The optimizations make your Rust code run faster, but turning them on lengthens the time it takes for your program to compile.最適化によってRustコードはより速く実行されますが、プログラムをコンパイルするのにかかる時間が長くなります。 This is why there are two different profiles: one for development, when you want to rebuild quickly and often, and another for building the final program you'll give to a user that won't be rebuilt repeatedly and that will run as fast as possible.これは、2つの異なるプロファイルがあります：1つは開発用、もう1つは迅速かつ頻繁に再構築するため、もう1つは繰り返しリビルドされないで、可能。 If you're benchmarking your code's running time, be sure to run cargo build --release and benchmark with the executable in target/release .コードの実行時間をベンチマークしている場合、 cargo build --releaseとbenchmarkをtarget / releaseの実行可能ファイルで実行してください。
