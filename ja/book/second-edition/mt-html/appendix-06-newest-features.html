Returning from loopsループからの戻り


One of the uses of a loop is to retry an operation you know can fail, such as checking if a thread completed its job. loopの使用法の1つは、スレッドがジョブを完了したかどうかを確認するなど、失敗する可能性のある操作を再試行することです。 However, you might need to pass the result of that operation to the rest of your code.ただし、その操作の結果をコードの残りの部分に渡す必要があるかもしれません。 If you add it to the break expression you use to stop the loop, it will be returned by the broken loop:ループを停止するために使用するbreak式に追加すると、ループが破損して戻されます。



Nested groups in use declarations use宣言のネストされたグループ


If you have a complex module tree with many different submodules and you need to import a few items from each one, it might be useful to group all the imports in the same declaration to keep your code clean and avoid repeating the base modules' name.多くの異なるサブモジュールを持つ複雑なモジュールツリーがあり、それぞれからいくつかの項目をインポートする必要がある場合は、同じ宣言ですべてのインポートをグループ化して、コードをきれいにしてベースモジュールの名前を繰り返さないようにすると便利です。

Inclusive ranges包括的範囲


Previously, when a range ( .. or ... ) was used as an expression, it had to be .. , which is exclusive of the upper bound, while patterns had to use ... , which is inclusive of the upper bound.以前は、範囲（ ..または... ）が式として使用されたときに、それは..でなければならず、上限は除外され、パターンは...を使用しなければならず、これは上限を含み... 。 Now, ..= is accepted as syntax for inclusive ranges in both expression and range context:今、 ..=は、式と範囲の両方のコンテキストで包括的な範囲の構文として受け入れられます：



The ... syntax is still accepted in matches, but it is not accepted in expressions. ...構文はマッチでは受け入れられますが、式では受け入れられません。 ..= should be preferred. ..=好ましいはず..= 。


128-bit integers 128ビット整数


Rust 1.26.0 added 128-bit integer primitives: Rust 1.26.0は128ビット整数プリミティブを追加しました：


u128 : A 128-bit unsigned integer with range 0, 2^128 - 1 u128 ：範囲が0, 2^128 - 1ビット符号なし整数
i128 : A 128-bit signed integer with range -(2^127), 2^127 - 1 i128 ：範囲が-(2^127), 2^127 - 1の128ビット符号付き整数

These primitives are implemented efficiently via LLVM support.これらのプリミティブは、LLVMサポートを使用して効率的に実装されます。 They are available even on platforms that don't natively support 128-bit integers and can be used like the other integer types.これらは、128ビット整数をネイティブにサポートしないプラットフォームでも利用でき、他の整数型と同様に使用できます。


These primitives can be very useful for algorithms that need to use very large integers efficiently, such as certain cryptographic algorithms.これらのプリミティブは、特定の暗号アルゴリズムなど、非常に大きな整数を効率的に使用する必要のあるアルゴリズムに非常に役立ちます。
