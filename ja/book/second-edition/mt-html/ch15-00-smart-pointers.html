Smart Pointersスマートポインタ


A pointer is a general concept for a variable that contains an address in memory. ポインタは、メモリ内のアドレスを含む変数の一般概念です。 This address refers to, or “points at,” some other data.このアドレスは、他のいくつかのデータを参照するか、または「ポイントする」。 The most common kind of pointer in Rust is a reference, which you learned about in Chapter 4. References are indicated by the & symbol and borrow the value they point to. Rustのポインタの最も一般的な種類は、第4章で学んだリファレンスです。参照は&記号で示され、それらが示す値を借ります。 They don't have any special capabilities other than referring to data.データを参照する以外に特別な機能はありません。 Also, they don't have any overhead and are the kind of pointer we use most often.また、それらにはオーバーヘッドがなく、私たちが最も頻繁に使用するポインタのようなものです。


Smart pointers , on the other hand, are data structures that not only act like a pointer but also have additional metadata and capabilities.一方、 スマートポインタは、ポインタのように動作するだけでなく、追加のメタデータと機能を持つデータ構造です。 The concept of smart pointers isn't unique to Rust: smart pointers originated in C++ and exist in other languages as well.スマートポインタの概念はRustにとってユニークではない：スマートポインタはC ++で作成され、他の言語でも存在する。 In Rust, the different smart pointers defined in the standard library provide functionality beyond that provided by references. Rustでは、標準ライブラリで定義されたさまざまなスマートポインタは、参照によって提供される機能を超えた機能を提供します。 One example that we'll explore in this chapter is the reference counting smart pointer type.この章で取り上げる1つの例は、 リファレンスカウントスマートポインタタイプです。 This pointer enables you to have multiple owners of data by keeping track of the number of owners and, when no owners remain, cleaning up the data.このポインタを使用すると、所有者の数を追跡し、所有者がいない場合はデータをクリーンアップすることによって、複数のデータ所有者を持つことができます。


In Rust, which uses the concept of ownership and borrowing, an additional difference between references and smart pointers is that references are pointers that only borrow data;所有権と借用の概念を使用するRustでは、参照とスマートポインタの追加の違いは、参照はデータを借りるだけのポインタであることです。 in contrast, in many cases, smart pointers own the data they point to.対照的に、多くの場合、スマートポインタはそれらが指すデータを所有しています。


We've already encountered a few smart pointers in this book, such as String and Vec<T> in Chapter 8, although we didn't call them smart pointers at the time.この本では、第8章のStringやVec<T>などのスマートポインタについては、すでにスマートポインタとは呼ばれていませんでしたが、ここでは遭遇しました。 Both these types count as smart pointers because they own some memory and allow you to manipulate it.これらの型はどちらもスマートポインタとしてカウントされます。これは、メモリを所有していて、操作できるようにするためです。 They also have metadata (such as their capacity) and extra capabilities or guarantees (such as with String ensuring its data will always be valid UTF-8).また、メタデータ（容量など）と追加の機能や保証（データが常に有効なUTF-8であることを保証するStringなど）もあります。


Smart pointers are usually implemented using structs.スマートポインタは、通常、構造体を使用して実装されます。 The characteristic that distinguishes a smart pointer from an ordinary struct is that smart pointers implement the Deref and Drop traits.通常の構造体からスマートポインタを区別する特性は、スマートポインタがDerefおよびDrop特性を実装することです。 The Deref trait allows an instance of the smart pointer struct to behave like a reference so you can write code that works with either references or smart pointers. Deref特徴は、スマートポインタ構造体のインスタンスを参照のように振る舞うことができるように、参照やスマートポインタのどちらでも動作するコードを書くことができます。 The Drop trait allows you to customize the code that is run when an instance of the smart pointer goes out of scope. Drop特性を使用すると、スマートポインタのインスタンスが有効範囲外になったときに実行されるコードをカスタマイズできます。 In this chapter, we'll discuss both traits and demonstrate why they're important to smart pointers.この章では、両方の特徴について説明し、それらがスマートポインタにとってなぜ重要であるかを説明します。


Given that the smart pointer pattern is a general design pattern used frequently in Rust, this chapter won't cover every existing smart pointer.スマートポインタパターンがRustで頻繁に使用される一般的なデザインパターンであることを考えると、この章では既存のスマートポインタはすべてカバーしません。 Many libraries have their own smart pointers, and you can even write your own.多くの図書館には独自のスマートポインタがあり、自分で書くこともできます。 We'll cover the most common smart pointers in the standard library:標準ライブラリで最も一般的なスマートポインタについて説明します。
