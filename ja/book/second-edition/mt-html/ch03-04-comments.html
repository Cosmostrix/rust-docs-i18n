Commentsコメント


All programmers strive to make their code easy to understand, but sometimes extra explanation is warranted.すべてのプログラマーは、コードを分かりやすくするために努力していますが、時には余分な説明が必要です。 In these cases, programmers leave notes, or comments , in their source code that the compiler will ignore but people reading the source code may find useful.このような場合、プログラマはコンパイラが無視するソースコードにメモやコメントを残しますが、ソースコードを読んでいる人は役に立つと思うかもしれません。


Here's a simple comment:ここに簡単なコメントがあります：


hello, worldこんにちは世界

In Rust, comments must start with two slashes and continue until the end of the line.錆では、コメントは2つのスラッシュで始まり、行の終わりまで続きます。 For comments that extend beyond a single line, you'll need to include // on each line, like this:単一行を越えるコメントについては、次のように各行に//を入れる必要があります。


So we're doing something complicated here, long enough that we need multiple lines of comments to do it!だから私たちはここで何か複雑な作業をしています。それを行うには、複数のコメント行が必要です。 Whew!すごい！ Hopefully, this comment will explain what's going on.うまくいけば、このコメントは何が起こっているのかを説明します。

Comments can also be placed at the end of lines containing code:コメントはコードを含む行の最後に置くこともできます：


Filename: src/main.rs ファイル名：src / main.rs


I'm feeling lucky today今日は運が良かった

But you'll more often see them used in this format, with the comment on a separate line above the code it's annotating:しかし、このフォーマットで使用されることがよくあります。注釈を付けるコードの上の別の行にコメントが表示されます。


Filename: src/main.rs ファイル名：src / main.rs


I'm feeling lucky today今日は運が良かった

Rust also has another kind of comment, documentation comments, which we'll discuss in Chapter 14. Rustには別の種類のコメント、ドキュメンテーションコメントがあります。これについては第14章で説明します。
