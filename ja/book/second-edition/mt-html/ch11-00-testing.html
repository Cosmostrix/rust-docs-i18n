Writing Automated Tests自動テストの作成


In his 1972 essay “The Humble Programmer,” Edsger W. Dijkstra said that “Program testing can be a very effective way to show the presence of bugs, but it is hopelessly inadequate for showing their absence.” That doesn't mean we shouldn't try to test as much as we can! Edsger W. Dijkstraは、1972年の彼のエッセイ「The Humble Programmer」では、「プログラムテストはバグの存在を示す非常に効果的な方法ですが、欠席を示すには絶望的です」と述べています。できるだけ多く試してみてください！


Correctness in our programs is the extent to which our code does what we intend it to do.私たちのプログラムの正しさは、私たちのコードが意図していることをどの程度行っているかです。 Rust is designed with a high degree of concern about the correctness of programs, but correctness is complex and not easy to prove.錆は、プログラムの正しさについて高い関心を持って設計されていますが、正確さは複雑であり、証明するのは容易ではありません。 Rust's type system shoulders a huge part of this burden, but the type system cannot catch every kind of incorrectness. Rustのタイプシステムはこの負担の大きな部分を負っていますが、タイプシステムはあらゆる種類の誤りをキャッチできません。 As such, Rust includes support for writing automated software tests within the language.そのため、Rustには言語内での自動ソフトウェアテストの作成がサポートされています。


As an example, say we write a function called add_two that adds 2 to whatever number is passed to it.例として、 add_twoという関数を書いて、渡される数値に2を加算するとします。 This function's signature accepts an integer as a parameter and returns an integer as a result.この関数のシグネチャは、パラメータとして整数を受け取り、結果として整数を返します。 When we implement and compile that function, Rust does all the type checking and borrow checking that you've learned so far to ensure that, for instance, we aren't passing a String value or an invalid reference to this function.この関数を実装してコンパイルするとき、Rustはこれまでに学んだすべての型チェックと借り入れチェックを行います。たとえば、 String値またはこの関数への無効な参照を渡さないようにします。 But Rust can't check that this function will do precisely what we intend, which is return the parameter plus 2 rather than, say, the parameter plus 10 or the parameter minus 50!しかし、Rust は 、この関数が意図したことを正確に行うことを確認することはできません 。例えば、パラメータプラス10やパラメータマイナス50ではなく、パラメータプラス2を返します。 That's where tests come in.それがテストの仕組みです。


We can write tests that assert, for example, that when we pass 3 to the add_two function, the returned value is 5 .たとえば、 add_two関数に3を渡したときに返される値は5あるとするテストを書くことができます。 We can run these tests whenever we make changes to our code to make sure any existing correct behavior has not changed.既存の正しい動作が変更されていないことを確認するために、コードを変更するたびにこれらのテストを実行できます。


Testing is a complex skill: although we can't cover every detail about how to write good tests in one chapter, we'll discuss the mechanics of Rust's testing facilities.テストは複雑なスキルです：1つの章で良いテストを書く方法のあらゆる詳細をカバーすることはできませんが、我々はRustのテスト機能のメカニズムについて議論します。 We'll talk about the annotations and macros available to you when writing your tests, the default behavior and options provided for running your tests, and how to organize tests into unit tests and integration tests.テストの作成時に使用できる注釈やマクロ、テストの実行に必要なデフォルトの動作とオプション、単体テストと統合テストへのテストの構成方法について説明します。
