Using Modules to Reuse and Organize Codeモジュールを使用したコードの再利用と整理


When you start writing programs in Rust, your code might live solely in the main function.あなたがRustでプログラムを書くのを始めるとき、あなたのコードはmain関数だけで生きるかもしれません。 As your code grows, you'll eventually move functionality into other functions for reuse and better organization.コードが大きくなるにつれ、最終的に機能を他の機能に移して再利用し、より良い組織を構築します。 By splitting your code into smaller chunks, you make each chunk easier to understand on its own.あなたのコードをより小さなチャンクに分割することで、各チャンクを独自に理解しやすくなります。 But what happens if you have too many functions?しかし、関数が多すぎるとどうなりますか？ Rust has a module system that enables the reuse of code in an organized fashion. Rustには、組織的な方法でコードの再利用を可能にするモジュールシステムがあります。


In the same way that you extract lines of code into a function, you can extract functions (and other code, like structs and enums) into different modules.関数のコード行を抽出するのと同じ方法で、関数（および構造体と列挙型のような他のコード）を別のモジュールに抽出することができます。 A module is a namespace that contains definitions of functions or types, and you can choose whether those definitions are visible outside their module (public) or not (private). モジュールは、関数や型の定義を含む名前空間であり、その定義がモジュールの外部（公開）か非外部（非公開）かを選択できます。 Here's an overview of how modules work:モジュールの仕組みの概要を以下に示します。


The mod keyword declares a new module. modキーワードは新しいモジュールを宣言します。 Code within the module appears either immediately following this declaration within curly brackets or in another file.モジュール内のコードは、この中括弧または他のファイルの宣言の直後に表示されます。
By default, functions, types, constants, and modules are private.デフォルトでは、関数、型、定数、およびモジュールはprivateです。 The pub keyword makes an item public and therefore visible outside its namespace. pubキーワードは、項目をパブリックにして、ネームスペースの外側に見えるようにします。
The use keyword brings modules, or the definitions inside modules, into scope so it's easier to refer to them. useキーワードは、モジュールまたはモジュール内の定義をスコープに入れて、参照するのが簡単です。
