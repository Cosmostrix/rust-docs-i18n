Patterns and Matchingパターンとマッチング


Patterns are a special syntax in Rust for matching against the structure of types, both complex and simple.パターンは複雑でシンプルなタイプの構造と照合するためのRustの特別な構文です。 Using patterns in conjunction with match expressions and other constructs gives you more control over a program's control flow.パターンをmatch式やその他の構文と共に使用すると、プログラムの制御フローをより詳細に制御できます。 A pattern consists of some combination of the following:パターンは、以下のいくつかの組み合わせで構成されます。


Literalsリテラル
Destructured arrays, enums, structs, or tuples構造化された配列、列挙型、構造体、またはタプル
Variables変数
Wildcardsワイルドカード
Placeholdersプレースホルダ

These components describe the shape of the data we're working with, which we then match against values to determine whether our program has the correct data to continue running a particular piece of code.これらのコンポーネントは、作業中のデータの形状を記述します。次に、値と照合して、プログラムが特定のコードを実行し続ける正しいデータを持っているかどうかを判断します。


To use a pattern, we compare it to some value.パターンを使用するには、それをある値と比較します。 If the pattern matches the value, we use the value parts in our code.パターンが値と一致する場合は、コード内の値の部分を使用します。 Recall the match expressions in Chapter 6 that used patterns, such as the coin-sorting machine example.第6章のコインソーティングマシンの例などのパターンを使用したmatch式を思い出してください。 If the value fits the shape of the pattern, we can use the named pieces.値がパターンの形に合っていれば、名前付きの部分を使用できます。 If it doesn't, the code associated with the pattern won't run.そうでない場合、パターンに関連付けられたコードは実行されません。


This chapter is a reference on all things related to patterns.この章では、パターンに関連するすべてのことを参照しています。 We'll cover the valid places to use patterns, the difference between refutable and irrefutable patterns, and the different kinds of pattern syntax that you might see.パターンを使用するための有効な場所、修正可能なパターンと反駁できないパターンの違い、および表示されるさまざまな種類のパターン構文について説明します。 By the end of the chapter, you'll know how to use patterns to express many concepts in a clear way.この章の最後では、パターンを使って多くの概念を明確な方法で表現する方法を知っています。
