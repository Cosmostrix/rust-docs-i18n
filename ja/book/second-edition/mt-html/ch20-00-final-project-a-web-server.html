Here is the plan to build the web server:ここでは、Webサーバーを構築する計画です：


Learn a bit about TCP and HTTP. TCPとHTTPについて少し学んでください。
Listen for TCP connections on a socket.ソケット上のTCP接続を待ち受けます。
Parse a small number of HTTP requests.少数のHTTPリクエストを解析します。
Create a proper HTTP response.適切なHTTP応答を作成します。
Improve the throughput of our server with a thread pool.スレッドプールを使用してサーバーのスループットを向上させます。

But before we get started, we should mention one detail: the method we'll use won't be the best way to build a web server with Rust.しかし、私たちが始める前に、私たちが使用する方法は、Rustを使ってWebサーバーを構築する最良の方法ではないという、一つの詳細について言及する必要があります。 A number of production-ready crates are available on https://crates.io/ that provide more complete web server and thread pool implementations than we'll build. https://crates.io/には、より完成度の高いWebサーバーとスレッドプールの実装を提供する多数のプロダクション対応のクレートが用意されています。


However, our intention in this chapter is to help you learn, not to take the easy route.しかし、この章の意図は、簡単なルートを取らないように、学習するのに役立つことです。 Because Rust is a systems programming language, we can choose the level of abstraction we want to work with and can go to a lower level than is possible or practical in other languages. Rustはシステムプログラミング言語であるため、作業したい抽象レベルを選択することができ、他の言語では可能でも実用的でもないレベルに下げることができます。 We'll write the basic HTTP server and thread pool manually so you can learn the general ideas and techniques behind the crates you might use in the future.基本的なHTTPサーバーとスレッドプールを手動で作成して、将来使用する可能性のある枠組みの一般的なアイデアやテクニックを学ぶことができます。
