Common Collections共通コレクション


Rust's standard library includes a number of very useful data structures called collections . Rustの標準ライブラリには、 コレクションと呼ばれる多数の非常に有用なデータ構造が含まれています 。 Most other data types represent one specific value, but collections can contain multiple values.他のほとんどのデータ型は1つの特定の値を表しますが、コレクションには複数の値を含めることができます。 Unlike the built-in array and tuple types, the data these collections point to is stored on the heap, which means the amount of data does not need to be known at compile time and can grow or shrink as the program runs.組み込みの配列型やタプル型とは異なり、これらのコレクションが指し示すデータはヒープに格納されるため、コンパイル時にデータの量を知る必要はなく、プログラムの実行に伴って拡大または縮小することができます。 Each kind of collection has different capabilities and costs, and choosing an appropriate one for your current situation is a skill you'll develop over time.各種類のコレクションには異なる機能とコストがあり、現在の状況に適したものを選択することは、時間の経過とともに発展するスキルです。 In this chapter, we'll discuss three collections that are used very often in Rust programs:この章では、Rustプログラムで頻繁に使用される3つのコレクションについて説明します。


A vector allows you to store a variable number of values next to each other. ベクトルを使用すると、可変数の値を互いに隣に格納することができます。
A string is a collection of characters. 文字列は文字の集合です。 We've mentioned the String type previously, but in this chapter we'll talk about it in depth.これまでString型について触れましたが、この章ではこれについて詳しく説明します。
A hash map allows you to associate a value with a particular key. ハッシュマップを使用すると、値を特定のキーに関連付けることができます。 It's a particular implementation of the more general data structure called a map .これは、 マップと呼ばれるより一般的なデータ構造の特定の実装です。

To learn about the other kinds of collections provided by the standard library, see the documentation .標準ライブラリが提供する他の種類のコレクションについては、ドキュメントを参照してください。
