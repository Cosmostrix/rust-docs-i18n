Object Oriented Programming Features of Rustオブジェクト指向プログラミングの錆の特徴


Object-oriented programming (OOP) is a way of modeling programs.オブジェクト指向プログラミング（OOP）は、プログラムをモデリングする方法です。 Objects came from Simula in the 1960s.オブジェクトは1960年代のSimulaから来たものです。 Those objects influenced Alan Kay's programming architecture in which objects pass messages to each other.これらのオブジェクトは、オブジェクトが互いにメッセージを渡すAlan Kayのプログラミングアーキテクチャに影響を与えました。 He coined the term object-oriented programming in 1967 to describe this architecture.彼はこのアーキテクチャを記述するために、1967年にオブジェクト指向プログラミングという用語を作り出しました。 Many competing definitions describe what OOP is;多くの競合する定義はOOPが何であるかを記述する。 some definitions would classify Rust as object oriented, but other definitions would not. Rustをオブジェクト指向として分類する定義もあれば、他の定義では定義しない定義もあります。 In this chapter, we'll explore certain characteristics that are commonly considered object oriented and how those characteristics translate to idiomatic Rust.この章では、一般にオブジェクト指向とみなされる特定の特性と、その特性がどのように慣用錆に変換されるかを探究します。 We'll then show you how to implement an object-oriented design pattern in Rust and discuss the trade-offs of doing so versus implementing a solution using some of Rust's strengths instead.次に、Rustにオブジェクト指向のデザインパターンを実装する方法を説明し、その代わりにRustの強みを使用してソリューションを実装する方法とのトレードオフについて説明します。
