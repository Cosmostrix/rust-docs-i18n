Enums and Pattern Matching列挙型とパターンマッチング


In this chapter we'll look at enumerations , also referred to as enums .この章では、我々はまた、 列挙型と呼ばれる、 列挙型を見てみましょう。 Enums allow you to define a type by enumerating its possible values.列挙型では、可能な値を列挙して型を定義できます。 First, we'll define and use an enum to show how an enum can encode meaning along with data.まず、列挙型を定義して使用し、列挙型がデータとともに意味をどのようにエンコードするかを示します。 Next, we'll explore a particularly useful enum, called Option , which expresses that a value can be either something or nothing.次に、 Optionと呼ばれる特に有用な列挙型について調べてみましょう。これは値が何かと何もないことを表現しています。 Then we'll look at how pattern matching in the match expression makes it easy to run different code for different values of an enum.次に、 match式のパターンマッチングによって、enumの異なる値に対して異なるコードを実行する方法を簡単に見ていきます。 Finally, we'll cover how the if let construct is another convenient and concise idiom available to you to handle enums in your code.最後に、 if let構造体が、コード内の列挙型を扱うために利用できるもう一つの便利で簡潔なイディオムであることを説明します。


Enums are a feature in many languages, but their capabilities differ in each language.列挙型は多くの言語の機能ですが、機能は言語ごとに異なります。 Rust's enums are most similar to algebraic data types in functional languages, such as F#, OCaml, and Haskell. Rustの列挙型は、F＃、OCaml、Haskellなどの関数型言語の代数データ型に最もよく似ています。
