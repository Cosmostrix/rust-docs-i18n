Release Channelsリリースチャンネル


The Rust project uses a concept called 'release channels' to manage releases. Rustプロジェクトはリリースを管理するために 'リリースチャンネル'という概念を使用しています。 It's important to understand this process to choose which version of Rust your project should use.このプロセスを理解して、プロジェクトで使用する必要がある錆のバージョンを選択することが重要です。


Overview概要


There are three channels for Rust releases: Rustリリースには3つのチャンネルがあります：


Nightlyナイトリー
Betaベータ
Stable安定

New nightly releases are created once a day.新しい夜間リリースは1日に1回作成されます。 Every six weeks, the latest nightly release is promoted to 'Beta'. 6週間ごとに、最新の夜間リリースが「ベータ版」に昇格されます。 At that point, it will only receive patches to fix serious errors.その時点では、重大なエラーを修正するためのパッチのみを受け取ります。 Six weeks later, the beta is promoted to 'Stable', and becomes the next release of 1.x . 6週間後、ベータ版は「安定版」に昇格し、 1.x次のリリースになります。


This process happens in parallel.このプロセスは並行して行われます。 So every six weeks, on the same day, nightly goes to beta, beta goes to stable.だから、6週間ごとに、同じ日に、毎晩ベータに行き、ベータは安定して行きます。 When 1.x is released, at the same time, 1.(x + 1)-beta is released, and the nightly becomes the first version of 1.(x + 2)-nightly . 1.xがリリースされると、同時に1.(x + 1)-betaが解放され、夜間は1の最初のバージョン1.(x + 2)-nightlyます。


Choosing a versionバージョンの選択


Generally speaking, unless you have a specific reason, you should be using the stable release channel.一般に、特定の理由がない限り、安定版リリースチャネルを使用する必要があります。 These releases are intended for a general audience.これらのリリースは、一般の読者を対象としています。


However, depending on your interest in Rust, you may choose to use nightly instead.しかし、あなたのRustへの関心によっては、代わりに夜間使用を選択することができます。 The basic trade-off is this: in the nightly channel, you can use unstable, new Rust features.基本的なトレードオフはこれです：夜間チャンネルでは、不安定で新しいRust機能を使用できます。 However, unstable features are subject to change, and so any new nightly release may break your code.しかし、不安定な機能は変更される可能性があるため、新しい夜間リリースはコードを壊す可能性があります。 If you use the stable release, you cannot use experimental features, but the next release of Rust will not cause significant issues through breaking changes.安定版リリースを使用している場合、実験的な機能は使用できませんが、Rustの次のリリースでは変更が破棄されても大きな問題は発生しません。


Helping the ecosystem through CI CIを通じてエコシステムを支援する


What about beta?ベータはどうですか？ We encourage all Rust users who use the stable release channel to also test against the beta channel in their continuous integration systems.安定版リリースチャネルを使用するすべてのRustユーザーは、継続的な統合システムでベータチャネルをテストすることをお勧めします。 This will help alert the team in case there's an accidental regression.これは、偶発的な回帰がある場合のためにチームに警告するのに役立ちます。


Additionally, testing against nightly can catch regressions even sooner, and so if you don't mind a third build, we'd appreciate testing against all channels.さらに、夜間のテストでは、より早く回帰を捉えることができます。したがって、3回目のビルドを気にしなければ、すべてのチャンネルに対してテストしていただければ幸いです。


As an example, many Rust programmers use Travis to test their crates, which is free for open source projects.一例として、多くのRustプログラマはTravisを使って自分の箱をテストしています。これはオープンソースプロジェクトでは無料です。 Travis supports Rust directly , and you can use a .travis.yml file like this to test on all channels: Travis はRustを直接サポートしています。このように.travis.ymlファイルを使用すると、すべてのチャンネルでテストできます：



travis

With this configuration, Travis will test all three channels, but if something breaks on nightly, it won't fail your build.この構成では、Travisは3つのチャンネルすべてをテストしますが、夜間に何か不具合が発生しても、ビルドは失敗しません。 A similar configuration is recommended for any CI system, check the documentation of the one you're using for more details.同様の構成がCIシステムに推奨されていますので、使用しているドキュメントの詳細を確認してください。
