Enums列挙型


An enum in Rust is a type that represents data that is one of several possible variants. Rustのenum型は、いくつかの可能なバリアントの1つであるデータを表す型です。 Each variant in the enum can optionally have data associated with it: enum型の各バリアントには、必要に応じて関連するデータがあります。



The syntax for defining variants resembles the syntaxes used to define structs: you can have variants with no data (like unit-like structs), variants with named data, and variants with unnamed data (like tuple structs).バリアントを定義するための構文は、構造体を定義するための構文に似ています。データがないバリアント（ユニットのような構造体）、名前付きデータのバリアント、名前のないデータのバリアント（タプル構造体など）を持つことができます。 Unlike separate struct definitions, however, an enum is a single type.ただし、別々の構造体定義とは異なり、 enum型は単一の型です。 A value of the enum can match any of the variants.列挙型の値は、任意のバリアントに一致することができます。 For this reason, an enum is sometimes called a 'sum type': the set of possible values of the enum is the sum of the sets of possible values for each variant.このため、列挙型は時には「合計型」と呼ばれます。列挙型の可能な値のセットは、各バリアントの可能な値のセットの合計です。


We use the :: syntax to use the name of each variant: they're scoped by the name of the enum itself. :: syntaxを使用して、各バリアントの名前を使用します。 enumは、 enum自体の名前でスコープされています。 This allows both of these to work:これにより、これらの両方を動作させることができます。

A value of an enum type contains information about which variant it is, in addition to any data associated with that variant. enum型の値には、そのバリアントに関連するデータに加えて、そのバリアントに関する情報が含まれます。 This is sometimes referred to as a 'tagged union', since the data includes a 'tag' indicating what type it is.データにはどのタイプのものであるかを示す「タグ」が含まれているため、これは「タグ付きユニオン」と呼ばれることがあります。 The compiler uses this information to enforce that you're accessing the data in the enum safely.コンパイラはこの情報を使用して、enum内のデータに安全にアクセスしていることを強制します。 For instance, you can't simply try to destructure a value as if it were one of the possible variants:たとえば、値が可能なバリアントの1つであるかのように、単に値を破棄することはできません。
