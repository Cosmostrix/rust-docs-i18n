Rust's main draw is its powerful static guarantees about behavior. Rustの主な魅力は、行動に対する強力な静的保証です。 But safety checks are conservative by nature: there are some programs that are actually safe, but the compiler is not able to verify this is true.しかし、安全性チェックは本質的に保守的です。実際には安全なプログラムがいくつかありますが、コンパイラはこれが正しいことを確認できません。 To write these kinds of programs, we need to tell the compiler to relax its restrictions a bit.この種のプログラムを書くためには、コンパイラに制限を緩和するよう伝える必要があります。 For this, Rust has a keyword, unsafe .このため、Rustにはキーワードがあり、 unsafe 。 Code using unsafe has fewer restrictions than normal code does. unsafeコードを使用すると、通常のコードより制限が少なくなります。

What does 'safe' mean? 「安全」とはどういう意味ですか？

Unsafe Superpowers安全でない超大国


In both unsafe functions and unsafe blocks, Rust will let you do three things that you normally can not do.安全でない機能と安全でないブロックの両方で、Rustは通常はできない3つのことをさせます。 Just three.ただ3つ。 Here they are:どうぞ：


Access or update a static mutable variable . 静的な可変変数にアクセスまたは更新します 。
Dereference a raw pointer.生のポインタを参照しない。
Call unsafe functions.安全でない関数を呼び出します。 This is the most powerful ability.これが最も強力な能力です。

That's it.それでおしまい。 It's important that unsafe does not, for example, 'turn off the borrow checker'. unsafeはないことが重要です。たとえば、「貸し出しチェッカーをオフにする」などです。 Adding unsafe to some random Rust code doesn't change its semantics, it won't start accepting anything.いくつかのランダムな錆のコードにunsafeを追加しても、セマンティクスは変わらず、何も受け入れられません。 But it will let you write things that do break some of the rules.しかし、それはあなたがルールのいくつかを破ることを書くことができます。


You will also encounter the unsafe keyword when writing bindings to foreign (non-Rust) interfaces.外部（非Rust）インターフェースへのバインディングを記述する際には、 unsafeキーワードが発生します。 You're encouraged to write a safe, native Rust interface around the methods provided by the library.ライブラリが提供するメソッドについて、安全でネイティブなRustインターフェースを書くことをお勧めします。

This last ability works with both aspects of unsafe : you can only call functions marked unsafe from inside an unsafe block.この最後の機能は、両方の側面で動作しunsafe ：あなたはマークされた関数のみを呼び出すことができますunsafe危険なブロックの内側から。


This ability is powerful and varied.この能力は強力で多様です。 Rust exposes some compiler intrinsics as unsafe functions, and some unsafe functions bypass safety checks, trading safety for speed. Rustはcompiler intrinsics関数の一部を安全でない関数として公開し、安全でない関数の一部はセーフティチェックをバイパスし、安全性を保証します。
