Unsized Typesサイズの小さい型


Most types have a particular size, in bytes, that is knowable at compile time.ほとんどの型はコンパイル時に知ることができる特定のサイズ（バイト単位）を持ちます。 For example, an i32 is thirty-two bits big, or four bytes.たとえば、 i32は32ビットビッグ、つまり4バイトです。 However, there are some types which are useful to express, but do not have a defined size.しかし、表現するのに便利ですが、定義されたサイズを持たないタイプもあります。 These are called 'unsized' or 'dynamically sized' types.これらは、「サイズ未定義」または「動的サイズ」タイプと呼ばれます。 One example is [T] .一例は[T]です。 This type represents a certain number of T in sequence.このタイプは、ある数のTを順番に表します。 But we don't know how many there are, so the size is not known.しかし、その数はわからないので、サイズは分かりません。


Rust understands a few of these types, but they have some restrictions.錆はこれらのタイプのいくつかを理解していますが、いくつかの制限があります。 There are three: 3つあります：


We can only manipulate an instance of an unsized type via a pointer.私たちは、ポインタを使ってunsized型のインスタンスを操作することしかできません。 An &[T] works fine, but a [T] does not. &[T]は正常に動作しますが、 [T]は動作しません。
Variables and arguments cannot have dynamically sized types.変数と引数には、動的にサイズの変更された型はありません。
Only the last field in a struct may have a dynamically sized type; struct最後のフィールドだけが動的にサイズの指定された型を持つことができます。 the other fields must not.他のフィールドは使用できません。 Enum variants must not have dynamically sized types as data. Enumバリアントは、データとして動的にサイズの変更された型を持つことはできません。
