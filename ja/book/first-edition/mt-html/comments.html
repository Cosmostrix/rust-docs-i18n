Commentsコメント


Now that we have some functions, it's a good idea to learn about comments.ここではいくつかの機能があるので、コメントについて学ぶのは良い考えです。 Comments are notes that you leave to other programmers to help explain things about your code.コメントは、他のプログラマーにあなたのコードに関することを説明するのに役立つメモです。 The compiler mostly ignores them.コンパイラはほとんど無視します。


Rust has two kinds of comments that you should care about: line comments and doc comments . Rustには、 行コメントと文書コメントの 2種類のコメントがあります。


Line comments are anything after '//' and extend to the end of the line.行コメントは '//'の後ろにあり、行の終わりまで拡張されます。
This is also a line comment.これもラインコメントです。
If you have a long explanation for something, you can put line comments next to each other.長い説明がある場合は、行のコメントを隣に置くことができます。 Put a space between the // and your comment so that it's more readable. //とあなたのコメントの間にスペースを入れて、読みやすくします。

The other kind of comment is a doc comment.他の種類のコメントはドキュメントコメントです。 Doc comments use /// instead of // , and support Markdown notation inside:ドキュメントコメントは、 //代わりに///を使用し、Markdown表記を内部でサポートします：


Adds one to the number given.与えられた数に1を加えます。
