Often, a simple if / else isn't enough, because you have more than two possible options.多くの場合、2つ以上のオプションがあるため、単純なif / elseでは不十分です。 Also, conditions can get quite complex.また、条件はかなり複雑になる可能性があります。 Rust has a keyword, match , that allows you to replace complicated if / else groupings with something more powerful. match takes an expression and then branches based on its value. matchは式をとり、その値に基づいて分岐します。 Each 'arm' of the branch is of the form val => expression .枝の各「腕」は、 val => expression形式です。 When the value matches, that arm's expression will be evaluated.値が一致すると、その腕の式が評価されます。 It's called match because of the term 'pattern matching', which match is an implementation of.それは呼ばれていますmatchので用語「パターンマッチング」のmatchの実装です。 There's a separate section on patterns that covers all the patterns that are possible here.ここで可能なすべてのパターンをカバーするパターンについての別のセクションがあります。

One of the many advantages of match is it enforces 'exhaustiveness checking'. matchの多くの利点の1つは、「徹底的なチェック」を実施することです。 For example if we remove the last arm with the underscore _ , the compiler will give us an error:たとえば、アンダースコア_ついた最後の腕を削除した場合、コンパイラはエラーを返します：



Rust is telling us that we forgot some value.錆は、我々が価値を忘れてしまったことを伝えている。 The compiler infers from x that it can have any 32bit integer value;コンパイラは、 xから任意の32ビット整数値を取り出すことができます。 for example -2,147,483,648 to 2,147,483,647.例えば、-2,147,483,648〜2,147,483,647である。 The _ acts as a 'catch-all', and will catch all possible values that aren't specified in an arm of match . _は 'catch-all'として機能し、 match腕に指定されていないすべての値を捕捉します。 As you can see in the previous example, we provide match arms for integers 1-5, if x is 6 or any other value, then it is caught by _ .前の例で分かるように、整数1〜5のmatchアームを提供しますxが6または他の値の場合、 _によってキャッチされます。


match is also an expression, which means we can use it on the right-hand side of a let binding or directly where an expression is used: matchは式でもあります。つまり、 letバインディングの右側で、または式が使用されている場所で直接使用できます。

Again, the Rust compiler checks exhaustiveness, so it demands that you have a match arm for every variant of the enum.繰り返しになりますが、Rustコンパイラは完全性をチェックするので、列挙型のすべての変形に対してマッチ・アームが必要です。 If you leave one off, it will give you a compile-time error unless you use _ or provide all possible arms. 1つをオフにすると、 _を使用したり、可能なすべてのアームを提供しない限り、コンパイル時エラーが発生します。


Unlike the previous uses of match , you can't use the normal if statement to do this.これまでのmatch使用とは異なり、通常のif文を使用することはできません。 You can use the if let statement, which can be seen as an abbreviated form of match . if let文を使用することができます。これは、 match省略形として見ることができます。
