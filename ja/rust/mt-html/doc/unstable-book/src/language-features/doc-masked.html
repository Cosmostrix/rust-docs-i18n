doc_masked

The doc_masked feature allows a crate to exclude types from a given crate from appearing in lists of trait implementations. doc_masked機能は、クレートが、特定のクレートのタイプを特性インプリメンテーションのリストから除外できるようにします。 The specifics of the feature are as follows:この機能の詳細は次のとおりです。


When rustdoc encounters an extern crate statement annotated with a #[doc(masked)] attribute, it marks the crate as being masked. rustdocが#[doc(masked)]属性でアノテーションされたextern crate crateステートメントに遭遇すると、クレートはマスクされたものとしてマークされます。

When listing traits a given type implements, rustdoc ensures that traits from masked crates are not emitted into the documentation.与えられた型が実装している特性をリストすると、rustdocはマスクされた木箱からの形質がドキュメントに出力されないようにします。

When listing types that implement a given trait, rustdoc ensures that types from masked crates are not emitted into the documentation.特定の特性を実装する型をリストするとき、rustdocはマスクされた木箱からの型がドキュメンテーションに放出されないようにします。

This feature was introduced in PR #44026 to ensure that compiler-internal and implementation-specific types and traits were not included in the standard library's documentation.この機能はPR #44026導入され、コンパイラ内部および実装固有の型と特性が標準ライブラリのドキュメントに含まれないようにしました。 Such types would introduce broken links into the documentation.そのようなタイプはドキュメントへの壊れたリンクを導入するでしょう。
