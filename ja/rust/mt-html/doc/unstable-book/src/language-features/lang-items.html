The rustc compiler has certain pluggable operations, that is, functionality that isn't hard-coded into the language, but is implemented in libraries, with a special marker to tell the compiler it exists. rustcコンパイラにはプラグイン可能な操作がいくつかあります。つまり、言語にハードコードされていないが、ライブラリに実装されているコンパイラが存在することをコンパイラに伝える特別なマーカーが付いた機能です。 The marker is the attribute #[lang = "..."] and there are various different values of ... , ie various different 'lang items'.マーカーは、属性である#[lang = "..."]との様々な異なる値が存在する...つまり、様々な異なる『LANG項目が』。

More about the language items言語項目の詳細


The compiler currently makes a few assumptions about symbols which are available in the executable to call.コンパイラは、現在、実行可能ファイル内で使用可能なシンボルについていくつかの仮定をしています。 Normally these functions are provided by the standard library, but without it you must define your own.通常、これらの関数は標準ライブラリによって提供されますが、それがなければ独自のライブラリを定義する必要があります。 These symbols are called "language items", and they each have an internal name, and then a signature that an implementation must conform to.これらの記号は「言語項目」と呼ばれ、それぞれが内部名を持ち、実装が準拠しなければならない署名を持っています。


The first of these functions, rust_eh_personality , is used by the failure mechanisms of the compiler.これらの関数のうちの最初のもの、 rust_eh_personalityは、コンパイラの失敗メカニズムによって使用されます。 This is often mapped to GCC's personality function (see the libstd implementation for more information), but crates which do not trigger a panic can be assured that this function is never called.これはしばしばGCCのパーソナリティ関数にマッピングされます（詳細についてはlibstd実装を参照してください）。しかし、パニックを引き起こさないクレートは、この関数が決して呼び出されないことを保証することができます。 The language item's name is eh_personality .言語項目の名前はeh_personalityです。


unwind

The second function, rust_begin_panic , is also used by the failure mechanisms of the compiler.第2の関数rust_begin_panicは、コンパイラの障害メカニズムによっても使用されます。 When a panic happens, this controls the message that's displayed on the screen.パニックが発生すると、これは画面に表示されるメッセージを制御します。 While the language item's name is panic_impl , the symbol name is rust_begin_panic .言語項目の名前はpanic_implが、シンボル名はrust_begin_panicです。


A third function, rust_eh_unwind_resume , is also needed if the custom_unwind_resume flag is set in the options of the compilation target.第3の関数rust_eh_unwind_resumeは、 custom_unwind_resumeフラグがコンパイル対象のオプションに設定されている場合にも必要です。 It allows customizing the process of resuming unwind at the end of the landing pads.着陸パッドの終わりに巻き戻すプロセスをカスタマイズすることができます。 The language item's name is eh_unwind_resume .言語項目の名前はeh_unwind_resumeです。
