This feature changes the way that "irrefutable patterns" are handled in the if let and while let forms.この機能は、「反駁可能なパターン」がif letおよびwhile letフォームで処理される方法を変更します。 An irrefutable pattern is one that cannot fail to match -- for example, the _ pattern matches any value, and hence it is "irrefutable". 反駁不能なパターンとは一致しないパターンです。たとえば、 _パターンは任意の値と一致します。したがって、 反駁できません。 Without this feature, using an irrefutable pattern in an if let gives a hard error (since often this indicates programmer error).この機能がないと、 if letで反駁不能なパターンを使用するとハードエラーが発生します（これはしばしばプログラマーエラーを示します）。 But when the feature is enabled, the error becomes a lint (since in some cases irrefutable patterns are expected).しかし、この機能が有効になっていると、エラーはリントになります（場合によっては反駁可能なパターンが予想されるため）。 This means you can use #[allow] to silence the lint:つまり、 #[allow]を使ってlintを消音することができます：
