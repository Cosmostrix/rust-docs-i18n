For extremely low-level manipulations and performance reasons, one might wish to control the CPU directly.極端に低レベルの操作やパフォーマンス上の理由から、CPUを直接制御したいかもしれません。 Rust supports using inline assembly to do this via the asm€ macro. Rustは、 asm€マクロを使用してインラインアセンブリを使用してこれをサポートします。



Any use of asm is feature gated (requires #€[feature(asm)] on the crate to allow) and of course requires an unsafe block. asm使用には機能があり#€[feature(asm)]クレートには#€[feature(asm)]が必要です）、もちろんunsafeブロックが必要です。

Some instructions modify registers which might otherwise have held different values so we use the clobbers list to indicate to the compiler not to assume any values loaded into those registers will stay valid.命令によっては、異なる値を保持する可能性のあるレジスタを変更するため、clobbersリストを使用して、レジスタにロードされた値が有効であると仮定しないことをコンパイラに指示します。

Input and output registers need not be listed since that information is already communicated by the given constraints.入力および出力レジスタは、その情報が所定の制約によって既に伝達されているので、リストされる必要はない。 Otherwise, any other registers used either implicitly or explicitly should be listed.それ以外の場合は、暗黙的にまたは明示的に使用される他のレジスタがリストされます。


If the assembly changes the condition code register cc should be specified as one of the clobbers.アセンブリが変更された場合、条件コードレジスタccをclobbersの1つとして指定する必要があります。 Similarly, if the assembly modifies memory, memory should also be specified.同様に、アセンブリがメモリを変更する場合は、 memoryも指定する必要があります。


Optionsオプション


The last section, options is specific to Rust.最後のセクション、 optionsはRust特有のものです。 The format is comma separated literal strings (ie :"foo", "bar", "baz" ).フォーマットはカンマで区切られたリテラル文字列です（例:"foo", "bar", "baz" ）。 It's used to specify some extra info about the inline assembly:これは、インラインアセンブリに関する追加情報を指定するために使用されます。


Current valid options are:現在有効なオプションは次のとおりです。


volatile - specifying this is analogous to __asm__ __volatile__ (...) in gcc/clang. volatile - これはgcc / clangの__asm__ __volatile__ (...)に似ています。
alignstack - certain instructions expect the stack to be aligned a certain way (ie SSE) and specifying this indicates to the compiler to insert its usual stack alignment code alignstack - 特定の命令は、スタックが特定の方法（つまりSSE）に揃えられることを期待し、これを指定すると、通常のスタックアライメントコードを挿入するようにコンパイラに指示します
intel - use intel syntax instead of the default AT&T. intel - デフォルトのAT＆Tではなくintel構文を使用します。
