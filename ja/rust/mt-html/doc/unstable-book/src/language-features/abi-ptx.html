When emitting PTX code, all vanilla Rust functions ( fn ) get translated to "device" functions. PTXコードを出力すると、すべてのvanilla Rust関数（ fn ）が "device"関数に変換されます。 These functions are not callable from the host via the CUDA API so a crate with only device functions is not too useful!これらの関数は、CUDA APIを介してホストから呼び出すことはできません。したがって、デバイス関数のみを持つクレートはそれほど有用ではありません。


OTOH, "global" functions can be called by the host; OTOHでは、 "グローバル"関数をホストから呼び出すことができます。 you can think of them as the real public API of your crate.それらをあなたのクレートの本当の公開APIと考えることができます。 To produce a global function use the "ptx-kernel" ABI.グローバル関数を生成するには、 "ptx-kernel" ABIを使用します。
