You can tell rustc how to customize linking, and that is via the link_args attribute.あなたはリンクをカスタマイズする方法をrustc伝えることができます。これはlink_args属性を使ってlink_argsます。 This attribute is applied to extern blocks and specifies raw flags which need to get passed to the linker when producing an artifact.この属性はexternブロックに適用され、アーティファクトを生成するときにリンカーに渡される必要がある生のフラグを指定します。 An example usage would be:使用例は次のとおりです。



Note that this feature is currently hidden behind the feature(link_args) gate because this is not a sanctioned way of performing linking.この機能は、 feature(link_args)ゲートの背後に隠されていることに注意してください。これは、承認されたリンクを実行する方法ではないためです。 Right now rustc shells out to the system linker ( gcc on most systems, link.exe on MSVC), so it makes sense to provide extra command line arguments, but this will not always be the case.現在、システムのリンカ（ほとんどのシステムではgcc 、MSVCではlink.exe ）にrustcシェルされているので、余分なコマンドライン引数を指定するのは理にかなっていますが、これは必ずしもlink.exeません。 In the future rustc may use LLVM directly to link native libraries, in which case link_args will have no meaning.将来、 rustcはLLVMを直接ネイティブライブラリにリンクすることがあります。この場合、 link_argsは意味を持ちません。 You can achieve the same effect as the link_args attribute with the -C link-args argument to rustc . link_argsの-C link-args引数をrustcすると、 link_args属性と同じ効果を得ることができます。


It is highly recommended to not use this attribute, and rather use the more formal #[link(...)] attribute on extern blocks instead.この属性を使用せず 、 externブロックでより正式な#[link(...)]属性を使用することを強くお勧めします。
