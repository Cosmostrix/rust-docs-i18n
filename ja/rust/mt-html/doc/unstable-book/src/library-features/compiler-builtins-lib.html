compiler_builtins_lib


The tracking issue for this feature is: None.この機能のトラッキングに関する問題は次のとおりです。


------------------------ ------------------------


This feature is required to link to the compiler_builtins crate which contains "compiler intrinsics".この機能は、コンパイラ組み込み関数を含むcompiler_builtins crateにリンクするために必要です。 Compiler intrinsics are software implementations of basic operations like multiplication of u64 s.コンパイラ組み込み関数は、 u64の乗算のような基本的な演算のソフトウェア実装です。 These intrinsics are only required on platforms where these operations don't directly map to a hardware instruction.これらの組み込み関数は、これらの操作がハードウェア命令に直接マッピングされないプラットフォームでのみ必要です。


You should never need to explicitly link to the compiler_builtins crate when building "std" programs as compiler_builtins is already in the dependency graph of std . compiler_builtinsが既にstd依存関係グラフにあるので、 "std"プログラムをビルドするときには、 compiler_builtinsパッケージに明示的にリンクする必要はありません。 But you may need it when building no_std binary crates.しかし、 no_std バイナリクレートをno_stdするときに必要になるかもしれません。 If you get a linker error like:次のようなリンカエラーが発生した場合：



That means that you need to link to this crate.つまり、この箱にリンクする必要があります。


When you link to this crate, make sure it only appears once in your crate dependency graph.このクレートにリンクするときは、クレート依存グラフに一度しか表示されていないことを確認してください。 Also, it doesn't matter where in the dependency graph you place the compiler_builtins crate.また、dependencyグラフのどこにcompiler_builtinsを配置するかは関係ありません。
