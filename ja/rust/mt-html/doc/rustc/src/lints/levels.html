Lint levels糸くずレベル


In rustc , lints are divided into four levels : rustc場合、糸くずは4つのレベルに分けられます ：


allow許す
warn警告する
deny否定する
forbid禁じる

Each lint has a default level (explained in the lint listing later in this chapter), and the compiler has a default warning level.各lintにはデフォルトレベル（この章の後半のlintリストで説明しています）があり、コンパイラにはデフォルトの警告レベルがあります。 First, let's explain what these levels mean, and then we'll talk about configuration.まず、これらのレベルが意味することを説明してから、構成について説明します。

deny否定する


A 'deny' lint produces an error if you violate it.あなたがそれに違反した場合、「拒否」リントはエラーを生成します。 For example, this code runs into the exceeding_bitshifts lint.たとえば、このコードはに実行さexceeding_bitshifts糸くず。




What's the difference between an error from a lint and a regular old error?糸くずのエラーと通常の古いエラーの違いは何ですか？ Lints are configurable via levels, so in a similar way to 'allow' lints, warnings that are 'deny' by default let you allow them.リントはレベルによって設定することができるので、lintと同様の方法で、デフォルトで '拒否'の警告が許可されます。 Similarly, you may wish to set up a lint that is warn by default to produce an error instead.同様に、デフォルトでwarnてエラーを生成するlintを設定することもできます。 This lint level gives you that.この糸くずレベルはあなたにそれを与えます。


forbid禁じる


'forbid' is a special lint level that's stronger than 'deny'. 'forbid'は、 '拒否'よりも強力な特別な糸くずレベルです。 It's the same as 'deny' in that a lint at this level will produce an error, but unlike the 'deny' level, the 'forbid' level can not be overridden to be anything lower than an error.このレベルの糸くずではエラーが発生するという点で '拒否'と同じですが、 '拒否'レベルとは異なり、 '禁止'レベルはエラーより低いレベルに上書きすることはできません。
