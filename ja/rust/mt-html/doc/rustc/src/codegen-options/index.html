Codegen options Codegenオプション


All of these options are passed to rustc via the -C flag, short for "codegen."これらのオプションはすべて、 "codegen"の略で-Cフラグを介してrustc渡されます。 You can see a version of this list for your exact compiler by running rustc -C help .正確なコンパイラのリストは、 rustc -C help実行することで確認できrustc -C help 。


ar ar


This option is deprecated and does nothing.このオプションは廃止され、何もしません。


linkerリンカ


This flag lets you control which linker rustc invokes to link your code.このフラグを使用すると、コードをリンクするために呼び出されるリンカーrustcを制御できます。


link-arg=val link-arg = val


This flag lets you append a single extra argument to the linker invocation.このフラグを使用すると、1つの追加の引数をリンカー呼び出しに追加できます。


"Append" is significant; "Append"は重要です。 you can pass this flag multiple times to add multiple arguments.このフラグを複数回渡して、複数の引数を追加することができます。


link-argsリンク引数


This flag lets you append multiple extra arguments to the linker invocation.このフラグを使用すると、複数の余分な引数をリンカ呼び出しに追加できます。 The options should be separated by spaces.オプションはスペースで区切る必要があります。


link-dead-codeリンクデッドコード


Normally, the linker will remove dead code.通常、リンカはデッドコードを削除します。 This flag disables this behavior.このフラグはこの動作を無効にします。


An example of when this flag might be useful is when trying to construct code coverage metrics.このフラグが役に立つ場合の例は、コードカバレッジメトリックを構築しようとするときです。


lto lto


This flag instructs LLVM to use link time optimizations .このフラグは、 リンク時間の最適化を使用するようにLLVMに指示します。


It takes one of two values, thin and fat .それはthinとfat 2つの値のいずれかをとります。 'thin' LTO is a new feature of LLVM , 'fat' referring to the classic version of LTO. 「薄い」LTO はLLVMの新機能であり 、古くからのLTOを指す「脂肪」です。


target-cpuターゲットCPU


This instructs rustc to generate code specifically for a particular processor.これは、特定のプロセッサ専用のコードを生成するようにrustcに指示します。


You can run rustc --print target-cpus to see the valid options to pass here. rustc --print target-cpusを実行すると、ここを通過する有効なオプションが表示されます。 Additionally, native can be passed to use the processor of the host machine.さらに、 nativeは、ホストマシンのプロセッサを使用するために渡すことができます。


target-featureターゲット機能


Individual targets will support different features;個々のターゲットはさまざまな機能をサポートします。 this flag lets you control enabling or disabling a feature.このフラグを使用すると、機能の有効化または無効化を制御できます。


To see the valid options and an example of use, run rustc --print target-features .有効なオプションと使用例を見るには、 rustc --print target-features実行してrustc --print target-features 。


passesパス


This flag can be used to add extra LLVM passes to the compilation.このフラグは、追加のLLVMパスをコンパイルに追加するために使用できます。


The list must be separated by spaces.リストはスペースで区切る必要があります。


llvm-args llvm-args


This flag can be used to pass a list of arguments directly to LLVM.このフラグを使用すると、引数のリストをLLVMに直接渡すことができます。


The list must be separated by spaces.リストはスペースで区切る必要があります。


save-temps save-temps


rustc will generate temporary files during compilation; rustcはコンパイル時に一時ファイルを生成します。 normally it will delete them after it's done with its work.通常は作業終了後に削除されます。 This option will cause them to be preserved instead of removed.このオプションは、削除されずに保存されます。


rpath rpath


This option allows you to set the value of rpath .このオプションを使用すると、 rpathの値を設定できます。


overflow-checksオーバーフローチェック


This flag allows you to control the behavior of integer overflow.このフラグを使用すると、整数オーバーフローの動作を制御できます。 This flag can be passed many options:このフラグには多くのオプションを渡すことができます：


To turn overflow checks on: y , yes , or on .オーバーフローチェックをオンにするには： y 、 yes 、またはon 。
To turn overflow checks off: n , no , or off .オーバーフローチェックをオフにする： n 、 no 、またはoff 。

no-prepopulate-passes事前移入なしパス


The pass manager comes pre-populated with a list of passes;パスマネージャーにはパスのリストがあらかじめ入力されています。 this flag ensures that list is empty.このフラグは、リストが空であることを保証します。


no-vectorize-loopsベクトル化しないループ


By default, rustc will attempt to vectorize loops .デフォルトでは、 rustcはvectorize loopsをvectorize loopsしようとしvectorize loops 。 This flag will turn that behavior off.このフラグはその動作をオフにします。


no-vectorize-slp no-vectorize-slp


By default, rustc will attempt to vectorize loops using superword-level parallelism .デフォルトでは、 rustcはrustc superword-level parallelismを使用してループをベクトル化しようとします。 This flag will turn that behavior off.このフラグはその動作をオフにします。


soft-floatソフトフロート


This option will make rustc generate code using "soft floats."このオプションは "soft floats"を使ってrustcがコードを生成するようにします。 By default, a lot of hardware supports floating point instructions, and so the code generated will take advantage of this.デフォルトでは、多くのハードウェアが浮動小数点命令をサポートしているため、生成されたコードはこれを利用します。 "soft floats" emulate floating point instructions in software. 「ソフトフロート」は、ソフトウェアで浮動小数点命令をエミュレートします。


prefer-dynamic動的優先


By default, rustc prefers to statically link dependencies.デフォルトでは、 rustcは依存関係を静的にリンクします。 This option will make it use dynamic linking instead.このオプションを使用すると、代わりに動的リンクが使用されます。


no-integrated-as統合されていない


LLVM comes with an internal assembler; LLVMには内部アセンブラが付属しています。 this option will let you use an external assembler instead.このオプションを使用すると、代わりに外部アセンブラを使用できます。


no-redzone無赤ゾーン


This flag allows you to disable the red zone ).このフラグを使用すると、赤いゾーンを無効にすることができます）。 This flag can be passed many options:このフラグには多くのオプションを渡すことができます：


To enable the red zone: y , yes , or on .赤いゾーンを有効にするには： y 、 yes 、またはon 。
To disable it: n , no , or off .これを無効にするにno 、 n 、 no 、またはoffます。

relocation-model再配置モデル


This option lets you choose which relocation model to use.このオプションを使用すると、使用する再配置モデルを選択できます。


To find the valid options for this flag, run rustc --print relocation-models .このフラグの有効なオプションを調べるには、 rustc --print relocation-models実行しrustc --print relocation-models 。


code-model=valコードモデル= val


This option lets you choose which code model to use.このオプションを使用すると、使用するコードモデルを選択できます。


To find the valid options for this flag, run rustc --print code-models .このフラグの有効なオプションを見つけるには、 rustc --print code-models実行しrustc --print code-models 。


metadataメタデータ


This option allows you to control the metadata used for symbol mangling.このオプションを使用すると、シンボルのマングリングに使用されるメタデータを制御できます。


extra-filename余分なファイル名


This option allows you to put extra data in each output filename.このオプションを使用すると、各出力ファイル名に余分なデータを置くことができます。


codegen-unitsコードゲンユニット


This flag lets you control how many threads are used when doing code generation.このフラグを使用すると、コード生成時に使用されるスレッドの数を制御できます。


Increasing paralellism may speed up compile times, but may also produce slower code.パラレル化を増やすとコンパイル時間が短縮されることがありますが、コードが遅くなることもあります。


remarkリマーク


This flag lets you print remarks for these optimization passes.このフラグを使用すると、これらの最適化パスの注釈を印刷できます。


The list of passes should be separated by spaces.パスのリストはスペースで区切る必要があります。


all will remark on every pass. allはすべてのパスで発言するでしょう。


no-stack-checkスタックチェックなし


This option is deprecated and does nothing.このオプションは廃止され、何もしません。


debuginfoデバッグ情報


This flag lets you control debug information:このフラグを使用すると、デバッグ情報を制御できます。


0 : no debug info at all 0 ：全くデバッグ情報なし
1 : line tables only 1 ：行テーブルのみ
2 : full debug info 2 ：完全なデバッグ情報

opt-levelオプトレベル


This flag lets you control the optimization level.このフラグを使用すると、最適化レベルを制御できます。


0 : no optimizations 0 ：最適化なし
1 : basic optimizations 1 ：基本的な最適化
2 : some optimizations 2 ：いくつかの最適化
3 : all optimizations 3 ：すべての最適化
s : optimize for binary size s ：バイナリサイズの最適化
z : optimize for binary size, but also turn off loop vectorization. z ：バイナリサイズを最適化するだけでなく、ループのベクトル化もオフにします。

debug-assertionsデバッグアサーション


This flag lets you turn cfg(debug_assertions) on or off.このフラグを使用すると、 cfg(debug_assertions)オンまたはオフにすることができます。


inline-thresholdインラインしきい値


This option lets you set the threshold for inlining a function.このオプションを使用すると、関数をインライン展開するためのしきい値を設定できます。


The default is 225.デフォルトは225です。


panicパニック


This option lets you control what happens when the code panics.このオプションを使用すると、コードがパニックするときの処理を制御できます。


abort : terminate the process upon panic abort ：パニック時にプロセスを終了する
unwind : unwind the stack upon panic unwind ：パニック時にスタックを巻き戻す

incremental増分


This flag allows you to enable incremental compilation.このフラグを使用すると、増分コンパイルを有効にすることができます。
