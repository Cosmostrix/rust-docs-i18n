Documenting platform-/feature-specific informationプラットフォーム/フィーチャー固有の情報の文書化


Because of the way Rustdoc documents a crate, the documentation it creates is specific to the target rustc compiles for. Rustdocがクレートを文書化する方法のために、それが作成するドキュメンテーションは、対象のrustcコンパイル用に固有のものです。 Anything that's specific to any other target is dropped via #[cfg] attribute processing early in the compilation process.他のターゲットに固有のものは、 #[cfg]属性処理を介してコンパイルプロセスの早い段階で削除されます。 However, Rustdoc has a trick up its sleeve to handle platform-specific code if it does receive it.それを受けない場合は、Rustdocは、プラットフォーム固有のコードを処理するために、その袖をトリックをアップしています。


Because Rustdoc doesn't need to fully compile a crate to binary, it replaces function bodies with loop {} to prevent having to process more than necessary. Rustdocはクレートをバイナリに完全にコンパイルする必要がないので、関数本体をloop {}に置き換えて、必要以上に処理する必要がなくなります。 This means that any code within a function that requires platform-specific pieces is ignored.つまり、プラットフォーム固有の部分を必要とする関数内のコードはすべて無視されます。 Combined with a special attribute, #[doc(cfg(...))] , you can tell Rustdoc exactly which platform something is supposed to run on, ensuring that doctests are only run on the appropriate platforms. #[doc(cfg(...))]という特別な属性と組み合わせることで、Rustdocに何かが実行されるはずのプラットフォームを正確に伝えることができ、doctestが適切なプラットフォームでのみ実行されるようにします。

This flag allows rustdoc to treat your rust code as the given edition.このフラグを指定すると、rustdocはあなたの腐食コードを指定のエディションとして扱うことができます。 It will compile doctests with the given edition as well.それは与えられた版でもdoctestsをコンパイルします。 As with rustc , the default edition that rustdoc will use is 2015 (the first edition). rustcとrustcに、 rustdocが使用するデフォルトのエディションは2015 （初版）です。
