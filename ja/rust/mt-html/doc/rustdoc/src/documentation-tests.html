Syntax reference構文リファレンス


The exact syntax for code blocks, including the edge cases, can be found in the Fenced Code Blocks section of the CommonMark specification.エッジケースを含むコードブロックの正確な構文は、CommonMark仕様のFenced Code Blocksセクションにあります。


Rustdoc also accepts indented code blocks as an alternative to fenced code blocks: instead of surrounding your code with three backticks, you can indent each line by four or more spaces. Rustdocはまた、 インデントされたコードブロックをフェンスのコードブロックの代わりに使用できます。コードを3つのバッククォートで囲む代わりに、各行を4つ以上のスペースでインデントすることができます。



These, too, are documented in the CommonMark specification, in the Indented Code Blocks section.これらも、CommonMark仕様の「 インデントされたコードブロック」セクションに記載されています。


However, it's preferable to use fenced code blocks over indented code blocks.ただし、インデントされたコードブロックには、囲まれたコードブロックを使用することをお勧めします。 Not only are fenced code blocks considered more idiomatic for Rust code, but there is no way to use directives such as ignore or should_panic with indented code blocks.フェンスされたコードブロックは錆コードに対してより慣用的であると見なされるだけでなく、インデントされたコードブロックでignoreやshould_panicなどのディレクティブを使用する方法はありません。
