The doc attribute has more options though! doc属性にはもっと多くのオプションがあります。 These don't involve the text of the output, but instead, various aspects of the presentation of the output.これらは出力のテキストを含まず、代わりに出力の提示のさまざまな側面を含む。 We've split them into two kinds below: attributes that are useful at the crate level, and ones that are useful at the item level.私たちは、それらをクレートレベルで有用な属性とアイテムレベルで有用な属性の2種類に分けました。

#[doc(hidden)]


Any item annotated with #[doc(hidden)] will not appear in the documentation, unless the strip-hidden pass is removed. #[doc(hidden)]注釈が付けられたアイテムは、 strip-hiddenパスが削除されない限り、ドキュメントには表示されません。


#[doc(primitive)]


Since primitive types are defined in the compiler, there's no place to attach documentation attributes.プリミティブ型はコンパイラで定義されているため、ドキュメント属性を添付する場所はありません。 This attribute is used by the standard library to provide a way to generate documentation for primitive types.この属性は、プリミティブ型のドキュメントを生成する方法を提供するために、標準ライブラリによって使用されます。
