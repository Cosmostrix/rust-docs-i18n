Command-line argumentsコマンドライン引数


Here's the list of arguments you can pass to rustdoc : rustdoc渡すことができる引数のリストはrustdocです：


-h / --help : help -h / --help ：help


Using this flag looks like this:このフラグを使用すると、次のようになります。



This will show rustdoc 's built-in help, which largely consists of a list of possible command-line flags.これはrustdocの組み込みヘルプを表示します。これは主に可能なコマンドラインフラグのリストで構成されています。


Some of rustdoc 's flags are unstable; rustdocのフラグのいくつかは不安定です。 this page only shows stable options, --help will show them all.このページには安定したオプションしか表示されません--helpがすべて表示します。


-V / --version : version information -V / --version ：バージョン情報


Using this flag looks like this:このフラグを使用すると、次のようになります。



This will show rustdoc 's version, which will look something like this:これはrustdocのバージョンを表示します。これは次のようになります：



-v / --verbose : more verbose output -v / --verbose ：より詳細な出力


Using this flag looks like this:このフラグを使用すると、次のようになります。



This enables "verbose mode", which means that more information will be written to standard out.これにより、 "詳細モード"が有効になります。これは、より多くの情報が標準出力に書き込まれることを意味します。 What is written depends on the other flags you've passed in. For example, with --version :書き込まれる内容は、渡された他のフラグに依存します。例えば、-- --version ：



-r / --input-format : input format -r / --input-format ：入力フォーマット


This flag is currently ignored;このフラグは現在無視されています。 the idea is that rustdoc would support various input formats, and you could specify them via this flag.その考え方は、 rustdocがさまざまな入力フォーマットをサポートすることであり、このフラグを使って指定することができます。


Rustdoc only supports Rust source code and Markdown input formats. Rustdocは、RustソースコードとMarkdown入力フォーマットのみをサポートします。 If the file ends in .md or .markdown , rustdoc treats it as a Markdown file.ファイルが.mdまたは.markdownで終わる場合、 rustdocはそれをMarkdownファイルとして扱います。 Otherwise, it assumes that the input file is Rust.それ以外の場合は、入力ファイルがRustであるとみなされます。



-w / --output-format : output format -w / --output-format ：出力フォーマット


This flag is currently ignored;このフラグは現在無視されています。 the idea is that rustdoc would support various output formats, and you could specify them via this flag.その考え方は、 rustdocがさまざまな出力フォーマットをサポートしていて、このフラグを使って指定できるということです。


Rustdoc only supports HTML output, and so this flag is redundant today. RustdocはHTML出力のみをサポートしているので、このフラグは今日は冗長です。


-o / --output : output path -o / --output ：出力パス


Using this flag looks like this:このフラグを使用すると、次のようになります。



By default, rustdoc 's output appears in a directory named doc in the current working directory.デフォルトでは、 rustdocの出力は現在の作業ディレクトリのdocというディレクトリに表示されます。 With this flag, it will place all output into the directory you specify.このフラグを指定すると、指定したディレクトリにすべての出力が配置されます。



--crate-name : controlling the name of the crate --crate-name ：クレートの名前を制御する


Using this flag looks like this:このフラグを使用すると、次のようになります。



By default, rustdoc assumes that the name of your crate is the same name as the .rs file.デフォルトでは、 rustdocは、あなたのクレートの名前が.rsファイルと同じ名前であることを前提としています。 --crate-name lets you override this assumption with whatever name you choose. --crate-nameあなたが選んだ名前でこの仮定を上書きすることができます。


-L / --library-path : where to look for dependencies -L / --library-path ：依存関係を探す場所


Using this flag looks like this:このフラグを使用すると、次のようになります。



If your crate has dependencies, rustdoc needs to know where to find them.あなたの木枠に依存関係がある場合、 rustdocはそれらを見つける場所を知る必要があります。 Passing --library-path gives rustdoc a list of places to look for these dependencies. --library-pathをrustdoc 、これらの依存関係を探す場所のリストがrustdoc与えられます。


This flag takes any number of directories as its argument, and will use all of them when searching.このフラグは、任意の数のディレクトリを引数としてとり、検索時にすべてのディレクトリを使用します。



--cfg : passing configuration flags --cfg ：設定フラグを渡す


Using this flag looks like this:このフラグを使用すると、次のようになります。



This flag accepts the same values as rustc --cfg , and uses it to configure compilation.このフラグは、 rustc --cfgと同じ値を受け取り、それを使ってコンパイルを設定します。 The example above uses feature , but any of the cfg values are acceptable.上記の例ではfeature使用していますが、 cfg値のいずれかを使用できます。


--extern : specify a dependency's location --extern ：依存関係の場所を指定する


Using this flag looks like this:このフラグを使用すると、次のようになります。



Similar to --library-path , --extern is about specifying the location of a dependency. --library-pathと同様に、-- --externは依存関係の場所の指定に関するものです。 --library-path provides directories to search in, --extern instead lets you specify exactly which dependency is located where. --library-pathは検索するディレクトリを提供しますが、代わりに--externでは依存関係がどこにあるかを正確に指定できます。


-C / --codegen : pass codegen options to rustc -C / --codegen ：codegenオプションをrustcに渡す


Using this flag looks like this:このフラグを使用すると、次のようになります。



When rustdoc generates documentation, looks for documentation tests, or executes documentation tests, it needs to compile some rust code, at least part-way. rustdocがドキュメンテーションを生成したり、ドキュメンテーション・テストを探したり、ドキュメンテーション・テストを実行したりするときは、少なくとも途中でいくつかの錆のコードをコンパイルする必要があります。 This flag allows you to tell rustdoc to provide some extra codegen options to rustc when it runs these compilations.このフラグは、rustdocに、これらのコンパイルを実行するときにrustcにいくつかの特別なcodegenオプションを提供するように指示します。 Most of the time, these options won't affect a regular documentation run, but if something depends on target features to be enabled, or documentation tests need to use some additional options, this flag allows you to affect that.ほとんどの場合、これらのオプションは通常のドキュメントの実行には影響しませんが、対象となる機能が有効になっているか、ドキュメントテストで追加オプションを使用する必要がある場合、このフラグを使用して影響を与えます。


The arguments to this flag are the same as those for the -C flag on rustc.このフラグの引数は、rustcの-Cフラグの引数と同じです。 Run rustc -C help to get the full list. rustc -C helpを実行して完全なリストを取得します。


--passes : add more rustdoc passes --passes ：rustdocパスを追加する


Using this flag looks like this:このフラグを使用すると、次のようになります。



An argument of "list" will print a list of possible "rustdoc passes", and other arguments will be the name of which passes to run in addition to the defaults.引数 "list"は可能な "rustdoc pass"のリストを出力し、他の引数はデフォルトに加えて実行するための名前になります。


For more details on passes, see the chapter on them .パスの詳細については、パスの章を参照してください 。


See also --no-defaults . --no-defaultsも参照してください。


--no-defaults : don't run default passes --no-defaults ：デフォルトパスを実行しない


Using this flag looks like this:このフラグを使用すると、次のようになります。



By default, rustdoc will run several passes over your code.デフォルトでは、 rustdocはコードに対していくつかのパスを実行します。 This removes those defaults, allowing you to use --passes to specify exactly which passes you want.これにより、これらのデフォルトが取り除かれ、 - --passesを使用して、必要なパスを正確に指定することができます。


For more details on passes, see the chapter on them .パスの詳細については、パスの章を参照してください 。


See also --passes . --passesも参照してください。


--test : run code examples as tests --test ：コード例をテストとして実行する


Using this flag looks like this:このフラグを使用すると、次のようになります。



This flag will run your code examples as tests.このフラグはコード例をテストとして実行します。 For more, see the chapter on documentation tests .詳細については、ドキュメントテストの章を参照してください。


See also --test-args . --test-argsも参照してください。


--test-args : pass options to test runner --test-args ：テストランナーにオプションを渡す


Using this flag looks like this:このフラグを使用すると、次のようになります。



This flag will pass options to the test runner when running documentation tests.このフラグは、ドキュメントテストを実行しているときにテストランナーにオプションを渡します。 For more, see the chapter on documentation tests .詳細については、ドキュメントテストの章を参照してください。


See also --test . --testも参照してください。


--target : generate documentation for the specified target triple --target ：指定されたターゲットのドキュメントを生成する


Using this flag looks like this:このフラグを使用すると、次のようになります。



Similar to the --target flag for rustc , this generates documentation for a target triple that's different than your host triple. rustcの--targetフラグと同様に、これはホストトリプルとは異なるターゲットトリプルのドキュメントを生成します。


All of the usual caveats of cross-compiling code apply.クロスコンパイルコードの通常の警告がすべて適用されます。

This flag takes a list of files, and inserts them into the <head> section of the rendered documentation.このフラグは、ファイルのリストを取得し、レンダリングされたドキュメントの<head>セクションに挿入します。

--markdown-playground-url : control the location of the playground --markdown-playground-url ： --markdown-playground-urlの位置を制御する


Using this flag looks like this:このフラグを使用すると、次のようになります。



When rendering a Markdown file, this flag gives the base URL of the Rust Playground, to use for generating Run buttons. Markdownファイルをレンダリングするとき、このフラグはRust PlaygroundのベースURLを与え、 Runボタンの生成に使用します。



--markdown-no-toc : don't generate a table of contents --markdown-no-toc ：目次を生成しない


Using this flag looks like this:このフラグを使用すると、次のようになります。



When generating documentation from a Markdown file, by default, rustdoc will generate a table of contents. Markdownファイルからドキュメントを生成する場合、デフォルトではrustdocはrustdocを生成します。 This flag suppresses that, and no TOC will be generated.このフラグはそれを抑止し、TOCは生成されません。



-e / --extend-css : extend rustdoc's CSS -e / --extend --extend-css ：extend rustdocのCSSを拡張する


Using this flag looks like this:このフラグを使用すると、次のようになります。



With this flag, the contents of the files you pass are included at the bottom of Rustdoc's theme.css file.このフラグでは、渡されたファイルの内容がRustdocのtheme.cssファイルの一番下に含まれています。


While this flag is stable, the contents of theme.css are not, so be careful!このフラグは安定していtheme.cssが、 theme.cssの内容は変更されていませんので注意してください！ Updates may break your theme extensions.アップデートによってテーマ拡張が壊れることがあります。


--sysroot : override the system root --sysroot ：システムルートを上書きする


Using this flag looks like this:このフラグを使用すると、次のようになります。



Similar to rustc --sysroot , this lets you change the sysroot rustdoc uses when compiling your code. rustc --sysrootと同様に、コードをコンパイルする際のrustdoc使用方法を変更することができます。
