The type inference engine is pretty smart.型推論エンジンはかなりスマートです。 It does more than looking at the type of the r-value during an initialization.初期化中にr-value型を調べる以上のことをします。 It also looks at how the variable is used afterwards to infer its type.また、変数が後でその型を推論するためにどのように使われるかを調べます。 Here's an advanced example of type inference:タイプ推論の高度な例を次に示します。


Because of the annotation, the compiler knows that elem has type u8.注釈のために、コンパイラはelemが型u8を持つことを知っていelem 。
Create an empty vector (a growable array).空のベクトル（拡張可能な配列）を作成します。
At this point the compiler doesn't know the exact type of vec , it just knows that it's a vector of something ( Vec<_> ).この時点で、コンパイラはvecの正確な型を知らず、何かのベクトル（ Vec<_> ）を知っているだけです。
Insert elem in the vector. elemをベクトルに挿入しelem 。
