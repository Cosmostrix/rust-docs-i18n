Channels


Rust provides asynchronous channels for communication between threads. Channels allow a unidirectional flow of information between two end-points: the Sender and the Receiver.


Channels have two endpoints: the Sender<T> and the Receiver<T>, where T is the type of the message to be transferred (type annotation is superfluous)
The sender endpoint can be copied
Each thread will send its id via the channel
The thread takes ownership over thread_tx Each thread queues a message in the channel
Sending is a non-blocking operation, the thread will continue immediately after sending its message
