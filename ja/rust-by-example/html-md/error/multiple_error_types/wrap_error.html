Wrapping errors


An alternative to boxing errors is to wrap them in your own error type.


We will defer to the parse error implementation for their error. Supplying extra info requires adding more data to the type.
This is a wrapper, so defer to the underlying types' implementation of fmt.
This already impls Error, so defer to its own implementation.
The cause is the underlying implementation error type. Is implicitly cast to the trait object &error::Error. This works because the underlying type already implements the Error trait.
Implement the conversion from ParseIntError to DoubleError. This will be automatically called by ? if a ParseIntError needs to be converted into a DoubleError.

This adds a bit more boilerplate for handling errors and might not be needed in all applications. There are some libraries that can take care of the boilerplate for you.
