Dependencies依存関係


crates.io is the Rust community's central package registry that serves as a location to discover and download packages. crates.ioは、Rustコミュニティのパッケージレジストリで、パッケージを発見してダウンロードする場所として機能します。 cargo is configured to use it by default to find requested packages. cargoは、要求されたパッケージを見つけるためにデフォルトでそれを使用するように構成されています。


To depend on a library hosted on crates.io , add it to your Cargo.toml .上でホストされているライブラリに依存するようにcrates.io 、あなたにそれを追加Cargo.toml 。


crates.io

Adding a dependency依存関係の追加


If your Cargo.toml doesn't already have a [dependencies] section, add that, then list the crate name and version that you would like to use.あなたのCargo.tomlまだ[dependencies]セクションがない場合は、それを追加し、使用したい木箱の名前とバージョンを表示してください。 This example adds a dependency of the time crate:この例では、 time crateの依存関係を追加します。



The version string is a semver version requirement.バージョン文字列はsemverバージョン要件です。 The specifying dependencies docs have more information about the options you have here. specifying dependenciesドキュメントには、ここにあるオプションに関する詳細があります。


semver

If we also wanted to add a dependency on the regex crate, we would not need to add [dependencies] for each crate listed. regex木枠に依存関係を追加する場合は、リストにある各木枠に[dependencies]を追加する必要はありません。 Here's what your whole Cargo.toml file would look like with dependencies on the time and regex crates:以下は、 Cargo.tomlファイル全体がtimeとregex木箱に依存するように見えるものです：



Re-run cargo build , and Cargo will fetch the new dependencies and all of their dependencies, compile them all, and update the Cargo.lock : cargo build再実行cargo buildと、Cargoは新しい依存関係とその依存関係をすべて取得し、すべてをコンパイルし、 Cargo.lockを更新しCargo.lock 。



Our Cargo.lock contains the exact information about which revision of all of these dependencies we used.私たちのCargo.lockは、これらの依存関係のすべてのリビジョンに関する正確な情報が含まれています。


Now, if regex gets updated, we will still build with the same revision until we choose to cargo update .今、 regexが更新されると、私たちはcargo updateを選択するまで、同じリビジョンでビルドします。


You can now use the regex library using extern crate in main.rs . main.rs extern crate main.rsを使用してregexライブラリを使用できるようにmain.rs 。
