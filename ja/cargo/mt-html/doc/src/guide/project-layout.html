Project Layoutプロジェクトのレイアウト


Cargo uses conventions for file placement to make it easy to dive into a new Cargo project: Cargoでは、新しいCargoプロジェクトへの参加を容易にするために、ファイル配置に関する規則を使用しています。



Cargo.toml and Cargo.lock are stored in the root of your project ( package root ). Cargo.tomlとCargo.lockは、プロジェクトのルート （ パッケージルート ）に格納されます。
Source code goes in the src directory.ソースコードはsrcディレクトリにあります。
The default library file is src/lib.rs .デフォルトのライブラリファイルはsrc/lib.rsです。
The default executable file is src/main.rs .デフォルトの実行可能ファイルはsrc/main.rsです。
Other executables can be placed in src/bin/*.rs .他の実行可能ファイルは、 src/bin/*.rs置くことができます。
Integration tests go in the tests directory (unit tests go in each file they're testing).統合テストはtestsディレクトリにありtests （単体テストはテストしている各ファイルにあります）。
Examples go in the examples directory.例はexamplesディレクトリにあります。
Benchmarks go in the benches directory.ベンチマークはbenchesディレクトリにあります。

These are explained in more detail in the manifest description .これらはmanifest descriptionでより詳細に説明されています。
