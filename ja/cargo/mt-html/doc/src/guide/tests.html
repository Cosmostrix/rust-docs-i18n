Testsテスト


Cargo can run your tests with the cargo test command.貨物は、 cargo testコマンドで検査を実行できます。 Cargo looks for tests to run in two places: in each of your src files and any tests in tests/ . Cargoは、各srcファイルとtests/内の2つの場所で実行するテストを探します。 Tests in your src files should be unit tests, and tests in tests/ should be integration-style tests. srcファイルのテストはtests/なければなりません。テストのテストは統合スタイルのテストでなければなりません。 As such, you'll need to import your crates into the files in tests .そのため、あなたの箱をtests中のファイルにインポートする必要があります。


Here's an example of running cargo test in our project, which currently has no tests:ここでは、プロジェクトでcargo testしていますが、現在のところテストはありません。



If our project had tests, we would see more output with the correct number of tests.私たちのプロジェクトにテストがあると、正しいテスト数でより多くの出力が得られます。


You can also run a specific test by passing a filter:フィルタを渡すことで、特定のテストを実行することもできます。



This will run any test with foo in its name.これは、その名前でfooを使ってテストを実行します。


cargo test runs additional checks as well. cargo testも追加チェックが行われます。 For example, it will compile any examples you've included and will also test the examples in your documentation.たとえば、含まれているサンプルをコンパイルし、ドキュメントのサンプルもテストします。 Please see the testing guide in the Rust documentation for more details.詳細については、Rustのマニュアルのテストガイドを参照してください。
