Once you're ready for release, you can use cargo build --release to compile your files with optimizations turned on:リリース準備が整ったら、 cargo build --releaseを使用して、最適化をオンにしてファイルをコンパイルできます。



cargo build --release puts the resulting binary in target/release instead of target/debug . cargo build --releaseは、結果のバイナリをtarget/debug代わりにtarget/releaseに置きます。


Compiling in debug mode is the default for development-- compilation time is shorter since the compiler doesn't do optimizations, but the code will run slower.デバッグモードでのコンパイルは、開発のデフォルトです。コンパイラは最適化を行わないため、コンパイル時間は短くなりますが、コードは遅く実行されます。 Release mode takes longer to compile, but the code will run faster.リリースモードはコンパイルに時間がかかりますが、コードはより速く実行されます。
