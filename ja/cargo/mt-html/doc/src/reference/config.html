Configuration構成


This document will explain how Cargo's configuration system works, as well as available keys or configuration.このドキュメントでは、Cargoの設定システムがどのように機能するか、利用可能なキーや設定について説明します。 For configuration of a project through its manifest, see the manifest format .マニフェストによるプロジェクトの設定については、 マニフェスト形式を参照してください。


Hierarchical structure階層構造



Cargo allows local configuration for a particular project as well as global configuration, like git. Cargoでは、gitのようなグローバルな設定だけでなく、特定のプロジェクトのローカル設定が可能です。 Cargo extends this to a hierarchical strategy.貨物はこれを階層的な戦略に拡張します。 If, for example, Cargo were invoked in /projects/foo/bar/baz , then the following configuration files would be probed for and unified in this order:たとえば、Cargoが/projects/foo/bar/bazで呼び出された場合、以下の設定ファイルが探査され、この順に統合されます。


/projects/foo/bar/baz/.cargo/config
/projects/foo/bar/.cargo/config
/projects/foo/.cargo/config
/projects/.cargo/config
/.cargo/config
$HOME/.cargo/config

With this structure, you can specify configuration per-project, and even possibly check it into version control.この構造では、プロジェクトごとの設定を指定し、場合によってはそれをバージョンコントロールにチェックすることもできます。 You can also specify personal defaults with a configuration file in your home directory.ホームディレクトリ内の設定ファイルで個人のデフォルトを指定することもできます。


Configuration format構成形式


All configuration is currently in the TOML format (like the manifest), with simple key-value pairs inside of sections (tables) which all get merged together.すべての構成は、現在マニフェストのようなTOML形式であり、すべてがマージされるセクション（テーブル）内に単純なキーと値のペアがあります。


toml

Configuration keys設定キー


All of the following keys are optional, and their defaults are listed as their value unless otherwise noted.次のキーはすべて省略可能です。特に指定がない限り、デフォルト値はその値としてリストされています。


Key values that specify a tool may be given as an absolute path, a relative path or as a pathless tool name.ツールを指定するキー値は、絶対パス、相対パス、またはパスレスツール名として与えられます。 Absolute paths and pathless tool names are used as given.与えられた絶対パスとパスレスツール名が使用されます。 Relative paths are resolved relative to the parent directory of the .cargo directory of the config file that the value resides within.相対パスは、その値が存在するconfigファイルの.cargoディレクトリの親ディレクトリを基準に解決されます。



Environment variables環境変数


Cargo can also be configured through environment variables in addition to the TOML syntax above.カーゴは、上記のTOML構文に加えて、環境変数を使用して構成することもできます。 For each configuration key above of the form foo.bar the environment variable CARGO_FOO_BAR can also be used to define the value.上記のfoo.barの各設定キーについて、環境変数CARGO_FOO_BARを使用して値を定義することもできます。 For example the build.jobs key can also be defined by CARGO_BUILD_JOBS .例えばbuild.jobsキーもによって定義することができCARGO_BUILD_JOBS 。


Environment variables will take precedent over TOML configuration, and currently only integer, boolean, and string keys are supported to be defined by environment variables.環境変数はTOML設定よりも先行し、現在は整数、ブール値、および文字列キーのみが環境変数で定義されています。 This means that source replacement , which is expressed by tables, cannot be configured through environment variables.つまり、テーブルで表されるソース置換は 、環境変数で設定することはできません。
