Package ID specificationsパッケージIDの仕様


Subcommands of Cargo frequently need to refer to a particular package within a dependency graph for various operations like updating, cleaning, building, etc. To solve this problem, Cargo supports Package ID Specifications. Cargoのサブコマンドは、更新、クリーニング、ビルドなどのさまざまな操作の依存関係グラフ内の特定のパッケージを頻繁に参照する必要があります。この問題を解決するために、CargoではパッケージID仕様がサポートされています。 A specification is a string which is used to uniquely refer to one package within a graph of packages.仕様は、パッケージのグラフ内の1つのパッケージを一意に参照するために使用される文字列です。

Brevity of specifications仕様の簡潔さ


The goal of this is to enable both succinct and exhaustive syntaxes for referring to packages in a dependency graph.これの目的は、依存関係グラフ内のパッケージを参照するための簡潔な構文と徹底的な構文の両方を有効にすることです。 Ambiguous references may refer to one or more packages.あいまいな参照は、1つまたは複数のパッケージを参照することがあります。 Most commands generate an error if more than one package could be referred to with the same specification.同じ仕様で複数のパッケージを参照できる場合、ほとんどのコマンドはエラーを生成します。
