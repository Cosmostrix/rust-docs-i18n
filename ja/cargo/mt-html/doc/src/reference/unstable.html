Unstable Features不安定な機能


Experimental Cargo features are only available on the nightly channel.実験的な貨物の機能は夜間のチャンネルでのみ利用できます。 You typically use one of the -Z flags to enable them.通常、それらを有効にするには、 -Zフラグの1つを使用します。 Run cargo -Z help to see a list of flags available. cargo -Z helpを実行すると、使用可能なフラグのリストが表示されます。


-Z unstable-options is a generic flag for enabling other unstable command-line flags. -Z unstable-optionsは、他の不安定なコマンドラインフラグを有効にするための汎用フラグです。 Options requiring this will be called out below.これを必要とするオプションについては以下で説明します。


Some unstable features will require you to specify the cargo-features key in Cargo.toml .いくつかの不安定な機能では、 Cargo.toml cargo-featuresキーを指定する必要があります。

Offline Modeオフラインモード

Original Issue: #4686元の問題： #4686
Tracking Issue: #5655トラッキングの問題： #5655

The -Z offline flag prevents Cargo from attempting to access the network for any reason. -Z offlineフラグは、何らかの理由でCargoがネットワークにアクセスしようとするのを防ぎます。 Typically Cargo will stop with an error if it wants to access the network and it is not available.通常、カーゴはネットワークにアクセスしたい場合にエラーで停止し、利用できません。


Beware that this may result in different dependency resolution than online mode.これにより、オンラインモードとは異なる依存関係が解決される可能性があることに注意してください。 Cargo will restrict itself to crates that are available locally, even if there might be a newer version as indicated in the local copy of the index. Cargoは、索引のローカルコピーに示されているように新しいバージョンが存在する場合でも、ローカルで使用可能なクレートに自身を制限します。


no-index-updateインデックスなしの更新

Original Issue: #3479元の問題： #3479

The -Z no-index-update flag ensures that Cargo does not attempt to update the registry index. -Z no-index-updateフラグは、Cargoがレジストリインデックスを更新しようとしないことを保証します。 This is intended for tools such as Crater that issue many Cargo commands, and you want to avoid the network latency for updating the index each time.これは、多くのCargoコマンドを発行するCraterなどのツールを対象としており、そのたびに索引を更新するためのネットワーク待ち時間を回避したいと考えています。


avoid-dev-deps避ける - dev-deps

When running commands such as cargo install or cargo build , Cargo currently requires dev-dependencies to be downloaded, even if they are not used. cargo installやcargo buildなどのコマンドを実行するとき、Cargoは現在、使用されていない場合でも、 cargo build者の依存関係をダウンロードする必要があります。 The -Z avoid-dev-deps flag allows Cargo to avoid downloading dev-dependencies if they are not needed. -Z avoid-dev-depsフラグは、Cargoが必要とされない場合、dev-dependenciesのダウンロードを避けることができます。 The Cargo.lock file will not be generated if dev-dependencies are skipped. dev-dependenciesをスキップすると、 Cargo.lockファイルは生成されません。


minimal-versions最小バージョン

Original Issue: #4100元の問題： #4100
Tracking Issue: #5657トラッキングの問題： #5657

When a Cargo.lock file is generated, the -Z minimal-versions flag will resolve the dependencies to the minimum semver version that will satisfy the requirements (instead of the greatest version). Cargo.lockファイルが生成されると、 -Z minimal-versionsフラグは依存関係を、（最大のバージョンではなく）要件を満たす-Z minimal-versionsに解決します。


The intended use-case of this flag is to check, during continuous integration, that the versions specified in Cargo.toml are a correct reflection of the minimum versions that you are actually using.このフラグの使用目的は、継続的な統合中にCargo.tomlで指定されたバージョンが、実際に使用している最小バージョンの正しいバージョンであることを確認することです。 That is, if Cargo.toml says foo = "1.0.0" that you don't accidentally depend on features added only in foo 1.5.0 .つまり、もしCargo.tomlがfoo = "1.0.0"と言っているのであれば、誤ってfoo 1.5.0だけ追加された機能に頼っているとは限りません。

To prevent unnecessary boilerplate from having to explicitly declare features for each optional dependency, implicit features get created for any optional dependencies where a feature of the same name is not defined.不要な定型文がオプションの依存関係ごとに明示的に機能を宣言するのを防ぐため、同じ名前の機能が定義されていないオプションの依存関係に対して暗黙の機能が作成されます。 However, if a feature of the same name as a dependency is defined, that feature must include the dependency as a requirement, as foo = ["crate:foo"] .ただし、依存関係と同じ名前の機能が定義されている場合、その機能はfoo = ["crate:foo"]ように依存関係を要件として含める必要があります。
