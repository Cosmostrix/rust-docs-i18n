Cargo supports the ability to replace one source with another to express strategies along the lines of mirrors or vendoring dependencies. With this configuration Cargo attempts to look up all crates in the directory "vendor" rather than querying the online registry at crates.io.この構成では、Cargoはcrates.ioのオンラインレジストリを照会するのではなく、ディレクトリ "vendor"のすべてのクレートを検索しようとします。 Using source replacement Cargo can express:ソース置換を使用する貨物は以下を表現できます：


Vendoring - custom sources can be defined which represent crates on the local filesystem. Vendoring - ローカルファイルシステム上のクレートを表すカスタムソースを定義することができます。 These sources are subsets of the source that they're replacing and can be checked into projects if necessary.これらのソースは、置き換え元のサブセットであり、必要に応じてプロジェクトにチェックインすることができます。

Mirroring - sources can be replaced with an equivalent version which acts as a cache for crates.io itself.ミラーリング - ソースは、crates.io自体のキャッシュとして機能する同等のバージョンに置き換えることができます。

Cargo has a core assumption about source replacement that the source code is exactly the same from both sources. Cargoは、ソースコードが両方のソースからまったく同じであるというソース置換についての中核的な前提を持っています。 In our above example Cargo assumes that all of the crates coming from my-awesome-source are the exact same as the copies from crates-io .上記の例では、Cargoはmy-awesome-sourceから来るすべての箱がcrates-ioからのコピーと全く同じであると仮定しています。 Note that this also means that my-awesome-source is not allowed to have crates which are not present in the crates-io source.これはまた、 my-awesome-sourceがcrates-ioソースに存在しないクレートを持つことが許可されていないことにも注意してください。


As a consequence, source replacement is not appropriate for situations such as patching a dependency or a private registry.結果として、ソース置換は、依存関係やプライベートレジストリのパッチ適用などの状況には適切ではありません。 Cargo supports patching dependencies through the usage of the [replace] key , and private registry support is planned for a future version of Cargo. Cargoは[replace]キーの使用による依存関係のパッチ適用をサポートしており、個人レジストリのサポートはCargoの将来のバージョンで計画されています。

Registry Sourcesレジストリソース


A "registry source" is one that is the same as crates.io itself. 「レジストリソース」は、crates.io自体と同じものです。 That is, it has an index served in a git repository which matches the format of the crates.io index .つまり、gitリポジトリには、 crates.ioインデックスの形式と一致するインデックスがあります 。 That repository then has configuration indicating where to download crates from.そのリポジトリには、どこからクレートをダウンロードするかを示す設定があります。


Currently there is not an already-available project for setting up a mirror of crates.io.現在、crates.ioのミラーを設定するための既に利用可能なプロジェクトはありません。 Stay tuned though!しかし、チューニングしておいてください！


Local Registry Sourcesローカルレジストリソース


A "local registry source" is intended to be a subset of another registry source, but available on the local filesystem (aka vendoring). "ローカルレジストリソース"は、別のレジストリソースのサブセットであることを意図していますが、ローカルファイルシステム（別名はベンダー）で利用できます。 Local registries are downloaded ahead of time, typically sync'd with a Cargo.lock , and are made up of a set of *.crate files and an index like the normal registry is.ローカルレジストリは、事前にダウンロードされます。一般的にCargo.lockとCargo.lockされ、 *.crateファイルのセットと通常のレジストリのようなインデックスで*.crateます。


The primary way to manage and crate local registry sources is through the cargo-local-registry subcommand, available on crates.io and can be installed with cargo install cargo-local-registry .ローカルのレジストリソースを管理および作成する主な方法は、crates.ioで利用可能なcargo-local-registryサブコマンドを使用し、 cargo install cargo-local-registryことができcargo install cargo-local-registry 。


cargo-local-registry

Local registries are contained within one directory and contain a number of *.crate files downloaded from crates.io as well as an index directory with the same format as the crates.io-index project (populated with just entries for the crates that are present).ローカルレジストリは1つのディレクトリに格納され、 *.crateからダウンロードされたいくつかの*.crateファイルと、crates.io-indexプロジェクトと同じフォーマットのindexディレクトリ（存在するクレートのエントリだけで構成される）。


Directory Sourcesディレクトリソース


A "directory source" is similar to a local registry source where it contains a number of crates available on the local filesystem, suitable for vendoring dependencies. "ディレクトリソース"はローカルレジストリソースに似ています。そこには、ローカルファイルシステム上で利用可能ないくつかのクレートが含まれています。 Also like local registries, directory sources can primarily be managed by an external subcommand, cargo-vendor , which can be installed with cargo install cargo-vendor .ローカルレジストリと同様に、ディレクトリソースは、主に外部サブコマンドであるcargo-vendorによって管理されます。これはcargo install cargo-vendorできます。


cargo-vendor

Directory sources are distinct from local registries though in that they contain the unpacked version of *.crate files, making it more suitable in some situations to check everything into source control.ディレクトリソースはローカルのレジストリとは異なりますが、アンパックされたバージョンの*.crateファイルが含まれているため、すべてをソース管理にチェックする状況に適しています。 A directory source is just a directory containing a number of other directories which contain the source code for crates (the unpacked version of *.crate files).ディレクトリソースは、クレートのソースコード（ *.crateファイルの*.crateていないバージョン）を含む多数の他のディレクトリを含むディレクトリです。 Currently no restriction is placed on the name of each directory.現時点では、各ディレクトリの名前に制限はありません。


Each crate in a directory source also has an associated metadata file indicating the checksum of each file in the crate to protect against accidental modifications.ディレクトリソース内の各クレートには、偶発的な変更から保護するために、クレート内の各ファイルのチェックサムを示す関連メタデータファイルもあります。
