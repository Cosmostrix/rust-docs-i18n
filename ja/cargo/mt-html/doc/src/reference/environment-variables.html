Environment Variables環境変数


Cargo sets and reads a number of environment variables which your code can detect or override. Cargoは、コードが検出または上書きできるいくつかの環境変数を設定し読み込みます。 Here is a list of the variables Cargo sets, organized by when it interacts with them: Cargoがそれらと相互作用するときに編成される変数のリストは次のとおりです。


Environment variables Cargo reads環境変数貨物読み込み


You can override these environment variables to change Cargo's behavior on your system:これらの環境変数をオーバーライドして、システム上のCargoの動作を変更することができます。


CARGO_HOME — Cargo maintains a local cache of the registry index and of git checkouts of crates. CARGO_HOME - Cargoは、レジストリインデックスとクレートのgit checkoutsのローカルキャッシュを管理しています。 By default these are stored under $HOME/.cargo , but this variable overrides the location of this directory.デフォルトでは、これらは$HOME/.cargo格納されますが、この変数はこのディレクトリの場所を上書きします。 Once a crate is cached it is not removed by the clean command.クレートがキャッシュされると、それはクリーンコマンドによって削除されません。
CARGO_TARGET_DIR — Location of where to place all generated artifacts, relative to the current working directory. CARGO_TARGET_DIR - 生成されたすべての成果物を現在の作業ディレクトリを基準に配置する場所。
RUSTC — Instead of running rustc , Cargo will execute this specified compiler instead. RUSTC - rustcを実行する代わりに、Cargoはこの指定されたコンパイラを代わりに実行します。
RUSTC_WRAPPER — Instead of simply running rustc , Cargo will execute this specified wrapper instead, passing as its commandline arguments the rustc invocation, with the first argument being rustc. RUSTC_WRAPPER - 単にrustcを実行する代わりに、Cargoは指定されたラッパーを代わりに実行し、コマンドライン引数としてrustc呼び出しを渡します。最初の引数はrustcです。
RUSTDOC — Instead of running rustdoc , Cargo will execute this specified rustdoc instance instead. RUSTDOC - rustdocを実行する代わりに、Cargoはこの指定されたrustdocインスタンスを代わりに実行します。
RUSTDOCFLAGS — A space-separated list of custom flags to pass to all rustdoc invocations that Cargo performs. RUSTDOCFLAGS - Cargoが実行するすべてのrustdoc呼び出しに渡すカスタムフラグのスペース区切りリスト。 In contrast with cargo rustdoc , this is useful for passing a flag to all rustdoc instances. cargo rustdocとは対照的に、これはすべての rustdocインスタンスにフラグを渡すのに便利です。
RUSTFLAGS — A space-separated list of custom flags to pass to all compiler invocations that Cargo performs. RUSTFLAGS - Cargoが実行するすべてのコンパイラ呼び出しに渡すカスタムフラグのスペース区切りリスト。 In contrast with cargo rustc , this is useful for passing a flag to all compiler instances. cargo rustcとは対照的に、これはすべてのコンパイラインスタンスにフラグを渡すのに便利です。
CARGO_INCREMENTAL — If this is set to 1 then Cargo will force incremental compilation to be enabled for the current compilation, and when set to 0 it will force disabling it. CARGO_INCREMENTAL - これを1に設定すると、Cargoは現在のコンパイルに対して増分コンパイルを有効にし、0に設定すると強制的に無効にします。 If this env var isn't present then cargo's defaults will otherwise be used.このenv varが存在しない場合は、貨物のデフォルトが使用されます。
CARGO_CACHE_RUSTC_INFO — If this is set to 0 then Cargo will not try to cache compiler version information. CARGO_CACHE_RUSTC_INFO - これを0に設定すると、Cargoはコンパイラのバージョン情報をキャッシュしようとしません。

Note that Cargo will also read environment variables for .cargo/config configuration values, as described in that documentation Cargoは、 そのドキュメントに記載されているように、 .cargo/config設定値の環境変数も読み込みます

CARGO - Path to the cargo binary performing the build. CARGO - ビルドを実行するcargoバイナリへのパス。
CARGO_MANIFEST_DIR - The directory containing the manifest for the package being built (the package containing the build script). CARGO_MANIFEST_DIR - ビルドされるパッケージ（ビルドスクリプトを含むパッケージ）のマニフェストを含むディレクトリ。 Also note that this is the value of the current working directory of the build script when it starts.また、これは、ビルドスクリプトが起動したときの現在の作業ディレクトリの値であることにも注意してください。
CARGO_MANIFEST_LINKS - the manifest links value. CARGO_MANIFEST_LINKS - マニフェストlinks値。
CARGO_FEATURE_<name> - For each activated feature of the package being built, this environment variable will be present where <name> is the name of the feature uppercased and having - translated to _ . CARGO_FEATURE_<name> -パッケージの各アクティブ化機能が内蔵されているため場合は、この環境変数が存在します<name>大文字とした機能の名前である-に翻訳_ 。
CARGO_CFG_<cfg> - For each configuration option of the package being built, this environment variable will contain the value of the configuration, where <cfg> is the name of the configuration uppercased and having - translated to _ . CARGO_CFG_<cfg> -ごとに設定オプションパッケージのが構築され、この環境変数を設定、の値が含まれています<cfg>大文字となる設定の名前である-に翻訳を_ 。 Boolean configurations are present if they are set, and not present otherwise.ブール構成は、設定されている場合は存在し、そうでない場合は存在しません。 Configurations with multiple values are joined to a single variable with the values delimited by , .複数の値を持つ構成はで区切られた値を持つ単一の変数に接合されています, 。
OUT_DIR - the folder in which all output should be placed. OUT_DIR - すべての出力を配置するフォルダ。 This folder is inside the build directory for the package being built, and it is unique for the package in question.このフォルダはビルド中のパッケージのビルドディレクトリの内側にあり、問題のパッケージで一意です。
TARGET - the target triple that is being compiled for. TARGET - コンパイルされるターゲットのトリプル。 Native code should be compiled for this triple.このトリプル用にネイティブコードをコンパイルする必要があります。 Some more information about target triples can be found in clang's own documentation .ターゲットのトリプルについての詳細は、 clang自身のドキュメントにあります 。
HOST - the host triple of the rust compiler. HOST - 錆びコンパイラのホストトリプル。
NUM_JOBS - the parallelism specified as the top-level parallelism. NUM_JOBS - 最上位の並列処理として指定された並列処理。 This can be useful to pass a -j parameter to a system like make .これはmakeようなシステムに-jパラメータを渡すのに便利です。 Note that care should be taken when interpreting this environment variable.この環境変数を解釈するときは注意が必要です。 For historical purposes this is still provided but recent versions of Cargo, for example, do not need to run make -j as it'll automatically happen.歴史的な目的のためにこれはまだ提供されていますが、最近のバージョンのCargoでは、自動的にmake -jを実行make -j必要はありません。 Cargo implements its own jobserver and will allow build scripts to inherit this information, so programs compatible with GNU make jobservers will already have appropriately configured parallelism. Cargoは独自のjobserverを実装し、ビルドスクリプトがこの情報を継承できるようにするので、GNU make jobserversと互換性のあるプログラムはすでに適切に並列化されています。
OPT_LEVEL , DEBUG - values of the corresponding variables for the profile currently being built. OPT_LEVEL 、 DEBUG - 現在構築中のプロファイルの対応する変数の値。
PROFILE - release for release builds, debug for other builds. PROFILE - releaseビルド用のリリース、他のビルド用のdebug
DEP_<name>_<key> - For more information about this set of environment variables, see build script documentation about links . DEP_<name>_<key> - この環境変数のセットの詳細については、 linksに関するビルドスクリプトのドキュメントを参照してください。
RUSTC , RUSTDOC - the compiler and documentation generator that Cargo has resolved to use, passed to the build script so it might use it as well. RUSTC 、 RUSTDOC - Cargoが使用することを決断したビルドスクリプトに渡されたコンパイラとドキュメントジェネレータです。
RUSTC_LINKER - The path to the linker binary that Cargo has resolved to use for the current target, if specified. RUSTC_LINKER - 指定されている場合、Cargoが現在のターゲットで使用するように解決したリンカーバイナリへのパス。 The linker can be changed by editing .cargo/config ;リンカーは.cargo/config編集することで変更できます。 see the documentation about cargo configuration for more information.詳細については、 貨物構成に関する文書を参照してください。
